> Prove that if we order the characters in an alphabet so that their frequencies are monotonically decreasing, then there exists an optimal code whose codeword lengths are monotonically increasing.

Suppose we have characters $$c_1, c_2, \ldots, c_n$$ with frequencies $$f_1 \ge f_2 \ge \cdots \ge f_n$$. We want to show there exists an optimal prefix code where the codeword lengths satisfy $$\ell_1 \le \ell_2 \le \cdots \le \ell_n$$.

Let $$T$$ be an optimal prefix code tree for these characters. We will show that if the codeword lengths are not monotonically increasing with decreasing frequency, we can rearrange the characters to achieve this property without increasing the cost.

Suppose for contradiction that in tree $$T$$, there exist characters $$c_i$$ and $$c_j$$ with $$f_i > f_j$$ but $$\ell_i > \ell_j$$ (the higher-frequency character has a longer codeword). Consider what happens if we swap the positions of $$c_i$$ and $$c_j$$ in the tree.

The cost before the swap includes the terms $$f_i \cdot \ell_i + f_j \cdot \ell_j$$. After swapping, the cost includes $$f_i \cdot \ell_j + f_j \cdot \ell_i$$. The change in cost is:

$$\Delta = (f_i \cdot \ell_j + f_j \cdot \ell_i) - (f_i \cdot \ell_i + f_j \cdot \ell_j)$$

Rearranging:

$$\Delta = f_i(\ell_j - \ell_i) + f_j(\ell_i - \ell_j) = (f_i - f_j)(\ell_j - \ell_i)$$

Since $$f_i > f_j$$, we have $$f_i - f_j > 0$$. Since $$\ell_i > \ell_j$$, we have $$\ell_j - \ell_i < 0$$. Therefore $$\Delta < 0$$, meaning the swap strictly decreases the cost.

But $$T$$ was assumed to be optimal, so no modification can decrease the cost. This is a contradiction. Therefore, in any optimal tree, if $$f_i > f_j$$, then $$\ell_i \le \ell_j$$.

For characters with equal frequencies, we can arrange them in any order at the same depth without affecting the cost. So we can always arrange an optimal code so that codeword lengths are monotonically increasing as frequencies decrease.

To complete the argument: starting from any optimal tree $$T$$, we can repeatedly swap any pair of characters that violates the monotonicity condition. Each swap either strictly decreases the cost (contradiction) or keeps the cost the same (when frequencies are equal). Since swaps that keep the cost the same still count as optimal, and the tree is finite, we can always reach an optimal arrangement where lengths increase monotonically with decreasing frequency.

{% capture note %}
This result is intuitive from a compression perspective: you want to use short codes for common characters and long codes for rare characters. The Huffman algorithm naturally produces codes with this property because it processes characters in order of increasing frequency, placing low-frequency characters deeper in the tree.
