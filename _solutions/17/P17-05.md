> ***Competitive analysis of self-organizing lists with move-to-front***
>
> A self-organizing list is a linked list of $$n$$ elements, in which each element has a unique key. When we search for an element in the list, we are given a key, and we want to find an element with that key.
>
> A self-organizing list has two important properties:
>
> 1. To find an element in the list, given its key, we must traverse the list from the beginning until we encounter the element with the given key. If that element is the $$k$$th element from the start of the list, then the cost to find the element is $$k$$.
>
> 2. We may reorder the list elements after any operation, according to a given rule with a given cost. We may choose any heuristic we like to decide how to reorder the list.
>
> Assume that we start with a given list of $$n$$ elements, and we are given an access sequence $$\sigma = \langle \sigma_1, \sigma_2, \ldots, \sigma_m \rangle$$ of keys to find, in order. The cost of the sequence is the sum of the costs of the individual accesses in the sequence.
>
> Out of the various possible ways to reorder the list after an operation, this problem focuses on transposing adjacent list elements (switching their positions in the list) with a unit cost for each transpose operation. You will show, by means of a potential function, that a particular heuristic for reordering the list, move-to-front, entails a total cost no worse than 4 times that of any other heuristic for maintaining the list order, even if the other heuristic knows the access sequence in advance! We call this type of analysis a competitive analysis.
>
> For a heuristic $$H$$ and a given initial ordering of the list, denote the access cost of sequence $$\sigma$$ by $$C_H(\sigma)$$. Let $$m$$ be the number of accesses in $$\sigma$$.
>
> 1. Argue that if heuristic $$H$$ does not know the access sequence in advance, then the worst-case cost for $$H$$ on an access sequence $$\sigma$$ is $$C_H(\sigma) = \Omega(mn)$$.
>
> With the move-to-front heuristic, immediately after searching for an element $$x$$, we move $$x$$ to the first position on the list (i.e., the front of the list).
>
> Let $$\text{rank}_L(x)$$ denote the rank of element $$x$$ in list $$L$$, that is, the position of $$x$$ in $$L$$. For example, if $$x$$ is the fourth element in $$L$$, then $$\text{rank}_L(x) = 4$$. Let $$c_i$$ denote the cost of access $$i$$ using the move-to-front heuristic, which includes the cost of finding the element in the list and the cost of moving it to the front of the list by a series of transpositions of adjacent list elements.
>
> 2. Show that if $$\sigma_i$$ accesses element $$x$$ in list $$L$$ using the move-to-front heuristic, then $$c_i = 2 \cdot \text{rank}_L(x) - 1$$.
>
> Now we compare move-to-front with any other heuristic $$H$$ that processes an access sequence according to the two properties above. Heuristic $$H$$ may transpose elements in the list in any way it wants, and it might even know the entire access sequence in advance.
>
> Let $$L_i$$ be the list after access $$i$$ using move-to-front, and let $$L_i^*$$ be the list after access $$i$$ using heuristic $$H$$. We denote the cost of access $$i$$ by $$c_i$$ for move-to-front and by $$c_i^*$$ for heuristic $$H$$. Suppose that heuristic $$H$$ performs $$t_i^*$$ transpositions during access $$i$$.
>
> 3. In part (b), you showed that $$c_i = 2 \cdot \text{rank}_{L_{i-1}}(x) - 1$$. Now show that $$c_i^* = \text{rank}_{L_{i-1}^*}(x) + t_i^*$$.
>
> We define an inversion in list $$L_i$$ as a pair of elements $$y$$ and $$z$$ such that $$y$$ precedes $$z$$ in $$L_i$$ and $$z$$ precedes $$y$$ in list $$L_i^*$$. Suppose that list $$L_i$$ has $$q_i$$ inversions after processing the access sequence $$\langle \sigma_1, \sigma_2, \ldots, \sigma_i \rangle$$. Then, we define a potential function $$\Phi$$ that maps $$L_i$$ to a real number by $$\Phi(L_i) = 2q_i$$. For example, if $$L_i$$ has the elements $$\langle e, c, a, d, b \rangle$$ and $$L_i^*$$ has the elements $$\langle c, a, b, d, e \rangle$$, then $$L_i$$ has 5 inversions ($$(e, c), (e, a), (e, d), (e, b), (d, b)$$), and so $$\Phi(L_i) = 10$$. Observe that $$\Phi(L_i) \ge 0$$ for all $$i$$ and that, if move-to-front and heuristic $$H$$ start with the same list $$L_0$$, then $$\Phi(L_0) = 0$$.
>
> 4. Argue that a transposition either increases the potential by 2 or decreases the potential by 2.
>
> Suppose that access $$i$$ finds the element $$x$$. To understand how the potential changes due to $$\sigma_i$$, let us partition the elements other than $$x$$ into four sets, depending on where they are in the lists just before the $$i$$th access:
>
> - Set $$A$$ consists of elements that precede $$x$$ in both $$L_{i-1}$$ and $$L_{i-1}^*$$.
> - Set $$B$$ consists of elements that precede $$x$$ in $$L_{i-1}$$ and follow $$x$$ in $$L_{i-1}^*$$.
> - Set $$C$$ consists of elements that follow $$x$$ in $$L_{i-1}$$ and precede $$x$$ in $$L_{i-1}^*$$.
> - Set $$D$$ consists of elements that follow $$x$$ in both $$L_{i-1}$$ and $$L_{i-1}^*$$.
>
> 5. Argue that $$\text{rank}_{L_{i-1}}(x) = |A| + |B| + 1$$ and $$\text{rank}_{L_{i-1}^*}(x) = |A| + |C| + 1$$.
>
> 6. Show that access $$i$$ causes a change in potential of $$\Phi(L_i) - \Phi(L_{i-1}) \le 2(|A| - |B| + t_i^*)$$, where, as before, heuristic $$H$$ performs $$t_i^*$$ transpositions during access $$i$$.
>
> Define the amortized cost $$\hat{c}_i$$ of access $$i$$ by $$\hat{c}_i = c_i + \Phi(L_i) - \Phi(L_{i-1})$$.
>
> 7. Show that the amortized cost $$\hat{c}_i$$ of access $$i$$ is bounded from above by $$4c_i^*$$.
>
> 8. Conclude that the cost $$C_{\text{MTF}}(\sigma)$$ of access sequence $$\sigma$$ with move-to-front is at most 4 times the cost $$C_H(\sigma)$$ of $$\sigma$$ with any other heuristic $$H$$, assuming that both heuristics start with the same list.

Competitive analysis compares an online algorithm (which processes requests one at a time without knowledge of future requests) against an optimal offline algorithm (which knows the entire request sequence in advance). The move-to-front heuristic is a classic example of an online algorithm with good competitive ratio. This problem shows that move-to-front is 4-competitive: its cost is at most 4 times the optimal, regardless of what access sequence an adversary chooses.

### A

An adversary can force any heuristic $$H$$ that does not know the access sequence to perform poorly. The strategy is simple: always request the element currently at the back of the list.

If $$H$$ does not know which element will be requested next, the adversary can always choose the element that $$H$$ has placed farthest from the front. Since there are $$n$$ elements, this element costs at least $$n$$ to access (it is at position $$n$$). Even if $$H$$ moves the accessed element closer to the front, the adversary simply requests a different element that is now at the back.

Over $$m$$ accesses, the adversary can ensure that at least a constant fraction of accesses cost $$\Omega(n)$$. More precisely, if $$H$$ does not rearrange the list, every access costs $$n$$, giving total cost $$mn$$. If $$H$$ does rearrange, the adversary adapts, but $$H$$ cannot anticipate which element will be requested. Thus, $$C_H(\sigma) = \Omega(mn)$$ in the worst case.

### B

When we access element $$x$$ at position $$k = \text{rank}_L(x)$$, two things happen:

1. **Finding $$x$$:** We traverse from the front of the list, examining $$k$$ elements. This costs $$k$$.

2. **Moving $$x$$ to front:** We perform $$k - 1$$ adjacent transpositions to move $$x$$ from position $$k$$ to position 1. Each transposition costs 1, so this costs $$k - 1$$.

The total cost is:

$$c_i = k + (k - 1) = 2k - 1 = 2 \cdot \text{rank}_L(x) - 1$$

{% capture note %}
The move-to-front heuristic is simple but not free: moving an element from position $$k$$ to the front costs nearly as much as finding it in the first place. However, this investment pays off if the element is accessed again soon.
{% endcapture %}

### C

For heuristic $$H$$, accessing element $$x$$ involves:

1. **Finding $$x$$:** We traverse from the front of $$H$$'s list $$L_{i-1}^*$$, examining $$\text{rank}_{L_{i-1}^*}(x)$$ elements. This costs $$\text{rank}_{L_{i-1}^*}(x)$$.

2. **Transpositions:** Heuristic $$H$$ may perform $$t_i^*$$ transpositions to rearrange the list after the access. Each transposition costs 1.

Therefore, the total cost for heuristic $$H$$ is:

$$c_i^* = \text{rank}_{L_{i-1}^*}(x) + t_i^*$$

### D

Consider a single transposition that swaps adjacent elements $$y$$ and $$z$$ in one of the lists (either $$L_i$$ or $$L_i^*$$). An inversion is a pair where the relative order differs between the two lists.

**Case 1:** Before the swap, $$y$$ and $$z$$ are in the same relative order in both lists. After swapping in one list, they are now in opposite orders. This creates a new inversion, increasing the count by 1, and the potential by 2.

**Case 2:** Before the swap, $$y$$ and $$z$$ are in opposite relative orders in the two lists (they form an inversion). After swapping in one list, they are now in the same relative order. This removes the inversion, decreasing the count by 1, and the potential by 2.

Since a transposition affects exactly one pair of adjacent elements, it either creates or destroys exactly one inversion. Therefore, each transposition changes the potential by exactly $$\pm 2$$.

### E

In list $$L_{i-1}$$, element $$x$$ is preceded by exactly $$\text{rank}_{L_{i-1}}(x) - 1$$ elements. These elements are partitioned into:
- Set $$A$$: elements that also precede $$x$$ in $$L_{i-1}^*$$
- Set $$B$$: elements that follow $$x$$ in $$L_{i-1}^*$$

Every element preceding $$x$$ in $$L_{i-1}$$ is in either $$A$$ or $$B$$ (not $$C$$ or $$D$$, since those follow $$x$$ in $$L_{i-1}$$). Therefore:

$$\text{rank}_{L_{i-1}}(x) - 1 = |A| + |B|$$

$$\text{rank}_{L_{i-1}}(x) = |A| + |B| + 1$$

Similarly, in list $$L_{i-1}^*$$, element $$x$$ is preceded by exactly $$\text{rank}_{L_{i-1}^*}(x) - 1$$ elements. These are partitioned into:
- Set $$A$$: elements that also precede $$x$$ in $$L_{i-1}$$
- Set $$C$$: elements that follow $$x$$ in $$L_{i-1}$$

Therefore:

$$\text{rank}_{L_{i-1}^*}(x) - 1 = |A| + |C|$$

$$\text{rank}_{L_{i-1}^*}(x) = |A| + |C| + 1$$

### F

After access $$i$$, the move-to-front heuristic moves $$x$$ to the front of $$L_i$$. Let us count how the inversions change.

**Inversions involving $$x$$:**

Before the access, consider pairs $$(x, y)$$ or $$(y, x)$$ where $$y \ne x$$.

- Elements in $$A$$: precede $$x$$ in both lists. No inversion with $$x$$.
- Elements in $$B$$: precede $$x$$ in $$L_{i-1}$$, follow $$x$$ in $$L_{i-1}^*$$. Each forms an inversion.
- Elements in $$C$$: follow $$x$$ in $$L_{i-1}$$, precede $$x$$ in $$L_{i-1}^*$$. Each forms an inversion.
- Elements in $$D$$: follow $$x$$ in both lists. No inversion with $$x$$.

After move-to-front, $$x$$ is first in $$L_i$$, so $$x$$ precedes all other elements in $$L_i$$.

- Elements in $$A$$: now follow $$x$$ in $$L_i$$, precede $$x$$ in $$L_i^*$$. Each creates an inversion ($$|A|$$ new inversions).
- Elements in $$B$$: now follow $$x$$ in $$L_i$$, follow $$x$$ in $$L_i^*$$ (assuming $$H$$ does not move $$x$$). No inversion. Previously there were $$|B|$$ inversions, now 0. ($$|B|$$ inversions removed)
- Elements in $$C$$: These followed $$x$$ in $$L_{i-1}$$ and preceded $$x$$ in $$L_{i-1}^*$$. After move-to-front, they still follow $$x$$ in $$L_i$$. If $$H$$ does not change $$x$$'s position, they still precede $$x$$ in $$L_i^*$$. Inversions remain.
- Elements in $$D$$: Still follow $$x$$ in both lists. No change to inversions involving $$x$$.

The net change from move-to-front (just considering inversions with $$x$$):

$$\Delta_{\text{MTF}} = |A| - |B|$$

**Effect of $$H$$'s transpositions:**

Heuristic $$H$$ performs $$t_i^*$$ transpositions. Each transposition can either increase or decrease the potential by 2. In the worst case for move-to-front, all of $$H$$'s transpositions increase the inversion count. This adds at most $$2t_i^*$$ to the potential.

**Total potential change:**

$$\Phi(L_i) - \Phi(L_{i-1}) \le 2(|A| - |B|) + 2t_i^* = 2(|A| - |B| + t_i^*)$$

### G

The amortized cost is:

$$\begin{align*}
\hat{c}_i &= c_i + \Phi(L_i) - \Phi(L_{i-1}) \\
&\le (2 \cdot \text{rank}_{L_{i-1}}(x) - 1) + 2(|A| - |B| + t_i^*)
\end{align*}$$

Substituting $$\text{rank}_{L_{i-1}}(x) = |A| + |B| + 1$$ from part E:

$$\begin{align*}
\hat{c}_i &\le 2(|A| + |B| + 1) - 1 + 2|A| - 2|B| + 2t_i^* \\
&= 2|A| + 2|B| + 2 - 1 + 2|A| - 2|B| + 2t_i^* \\
&= 4|A| + 1 + 2t_i^*
\end{align*}$$

From part E, $$\text{rank}_{L_{i-1}^*}(x) = |A| + |C| + 1$$, so $$|A| \le \text{rank}_{L_{i-1}^*}(x) - 1$$. Thus:

$$\begin{align*}
\hat{c}_i &\le 4(|A| + 1) + 2t_i^* - 3 \\
&\le 4(\text{rank}_{L_{i-1}^*}(x)) + 2t_i^* - 3
\end{align*}$$

Wait, let me be more careful. We have $$|A| \le |A| + |C| = \text{rank}_{L_{i-1}^*}(x) - 1$$. So:

$$\begin{align*}
\hat{c}_i &\le 4|A| + 1 + 2t_i^* \\
&\le 4(\text{rank}_{L_{i-1}^*}(x) - 1) + 1 + 2t_i^* \\
&= 4 \cdot \text{rank}_{L_{i-1}^*}(x) - 4 + 1 + 2t_i^* \\
&= 4 \cdot \text{rank}_{L_{i-1}^*}(x) - 3 + 2t_i^*
\end{align*}$$

From part C, $$c_i^* = \text{rank}_{L_{i-1}^*}(x) + t_i^*$$. We want to show $$\hat{c}_i \le 4c_i^*$$:

$$\begin{align*}
4c_i^* &= 4 \cdot \text{rank}_{L_{i-1}^*}(x) + 4t_i^*
\end{align*}$$

We have:

$$\begin{align*}
\hat{c}_i &\le 4 \cdot \text{rank}_{L_{i-1}^*}(x) - 3 + 2t_i^* \\
&\le 4 \cdot \text{rank}_{L_{i-1}^*}(x) + 4t_i^* \\
&= 4c_i^*
\end{align*}$$

The inequality holds because $$-3 + 2t_i^* \le 4t_i^*$$ (since $$t_i^* \ge 0$$, we have $$2t_i^* \ge 0 > -3$$, so $$-3 + 2t_i^* < 4t_i^*$$).

Therefore, $$\hat{c}_i \le 4c_i^*$$.

### H

Summing over all $$m$$ accesses:

$$\begin{align*}
C_{\text{MTF}}(\sigma) &= \sum_{i=1}^{m} c_i \\
&= \sum_{i=1}^{m} \hat{c}_i - \Phi(L_m) + \Phi(L_0) \\
&= \sum_{i=1}^{m} \hat{c}_i - \Phi(L_m)
\end{align*}$$

The last equality holds because $$\Phi(L_0) = 0$$ (both lists start identical, so there are no inversions).

Since $$\Phi(L_m) \ge 0$$:

$$\begin{align*}
C_{\text{MTF}}(\sigma) &\le \sum_{i=1}^{m} \hat{c}_i \\
&\le \sum_{i=1}^{m} 4c_i^* \\
&= 4 \sum_{i=1}^{m} c_i^* \\
&= 4 \cdot C_H(\sigma)
\end{align*}$$

Therefore, $$C_{\text{MTF}}(\sigma) \le 4 \cdot C_H(\sigma)$$ for any heuristic $$H$$ starting from the same initial list.

{% capture note %}
This result is remarkable: move-to-front, a simple online algorithm with no knowledge of future accesses, is within a factor of 4 of any algorithm, including one with perfect knowledge of the entire access sequence. The chapter notes mention that if we allow "free" moves (splicing rather than transposing), the competitive ratio improves to 2.
{% endcapture %}
{% include aside.html title='The power of competitive analysis' %}

This competitive ratio of 4 is tight in the sense that there exist access sequences where move-to-front does cost close to 4 times the optimal. However, in practice, move-to-front often performs much better, especially when accesses exhibit temporal locality (recently accessed items are likely to be accessed again soon).
