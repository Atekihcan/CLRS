> ***Viterbi algorithm***
>
> We can use dynamic programming on a directed graph $$G = (V, E)$$ for speech recognition. Each edge $$(u, v) \in E$$ is labeled with a sound $$\sigma(u, v)$$ from a finite set $$\Sigma$$ of sounds. The labeled graph is a formal model of a person speaking a restricted language. Each path in the graph starting from a distinguished vertex $$v_0 \in V$$ corresponds to a possible sequence of sounds produced by the model. We define the label of a directed path to be the concatenation of the labels of the edges on that path.
>
> 1. Describe an efficient algorithm that, given an edge-labeled graph $$G$$ with distinguished vertex $$v_0$$ and a sequence $$s = \langle\sigma_1, \sigma_2, \ldots, \sigma_k\rangle$$ of sounds from $$\Sigma$$, returns a path in $$G$$ that begins at $$v_0$$ and has $$s$$ as its label, if any such path exists. Otherwise, the algorithm should return NO-SUCH-PATH. Analyze the running time of your algorithm. (Hint: You may find concepts from Chapter 22 useful.)
>
> 2. Now, suppose that every edge $$(u, v) \in E$$ has an associated nonnegative probability $$p(u, v)$$ of traversing the edge $$(u, v)$$ from vertex $$u$$ and thus producing the corresponding sound. The sum of the probabilities of the edges leaving any vertex equals 1. The probability of a path is defined to be the product of the probabilities of its edges. Extend your answer to part (a) so that if a path is returned, it is a most probable path starting at $$v_0$$ and having label $$s$$. Analyze the running time of your algorithm.

The [Viterbi algorithm](https://en.wikipedia.org/wiki/Viterbi_algorithm){:target="_blank"} is one of the most important algorithms in signal processing and machine learning. It's used in speech recognition, DNA sequence analysis, and many other applications where we need to find the most likely sequence of hidden states given observed outputs.

### A. Finding Any Matching Path

We want to find if there's any path from $$v_0$$ that produces the sound sequence $$s = \langle\sigma_1, \sigma_2, \ldots, \sigma_k\rangle$$.

Think of it as a reachability problem in layers. After reading $$i$$ sounds, we want to know which vertices are reachable. This is similar to BFS, but we're following only edges whose labels match the next required sound.

Let $$R[i]$$ be the set of vertices reachable from $$v_0$$ by following a path whose label is $$\langle\sigma_1, \sigma_2, \ldots, \sigma_i\rangle$$.

**Base case:** $$R[0] = \{v_0\}$$

**Recurrence:** $$R[i] = \{ v : \exists u \in R[i-1] \text{ such that } (u, v) \in E \text{ and } \sigma(u, v) = \sigma_i \}$$

If $$R[k]$$ is non-empty after processing all $$k$$ sounds, we have a valid path.

{% include ads.html %}

{% capture code %}
Find-Path(G, v0, s, k)
    // Initialize
    let R[0..k] be new sets
    let prev[0..k, V] be a new table    // for path reconstruction
    R[0] = {v0}

    // Process each sound
    for i = 1 to k
        R[i] = empty set
        for each vertex u in R[i-1]
            for each edge (u, v) in Adj[u]
                if sigma(u, v) == s[i]
                    add v to R[i]
                    prev[i][v] = u      // remember how we got here

    // Check for valid path
    if R[k] is empty
        return NO-SUCH-PATH

    // Reconstruct path (pick any vertex in R[k])
    v = any vertex in R[k]
    path = empty list
    for i = k downto 1
        prepend v to path
        v = prev[i][v]
    prepend v0 to path

    return path
{% endcapture %}
{% include clrs_code.html title="Find-Path(G, v_0, s, k)" %}

**Running Time:** For each of the $$k$$ sounds, we potentially examine every edge in the graph. In the worst case, this gives $$O(k \cdot E)$$. If we use hash tables for the sets $$R[i]$$, set operations are $$O(1)$$ expected time.

More precisely, if we process sound $$i$$, we examine only edges leaving vertices in $$R[i-1]$$. Across all iterations, each vertex can appear in each $$R[i]$$, so the total work is $$O(k \cdot |V| \cdot \text{max-out-degree})$$, which is $$O(k \cdot E)$$ in the worst case.

**Space:** $$O(k \cdot V)$$ for the prev table and the sets.

### B. Finding the Most Probable Path

Now each edge has a probability, and we want the path with maximum probability. The key insight is that products of probabilities become sums when we take logarithms, so maximizing probability is equivalent to maximizing the sum of log-probabilities.

However, we can work directly with probabilities using DP.

Let $$P[i][v]$$ be the maximum probability of any path from $$v_0$$ to $$v$$ whose label is $$\langle\sigma_1, \sigma_2, \ldots, \sigma_i\rangle$$.

**Base case:** $$P[0][v_0] = 1$$, and $$P[0][v] = 0$$ for $$v \neq v_0$$

**Recurrence:**

$$P[i][v] = \max_{(u, v) \in E, \sigma(u,v) = \sigma_i} \{ P[i-1][u] \cdot p(u, v) \}$$

{% capture code %}
Viterbi(G, v0, s, k)
    // Initialize
    let P[0..k, V] be a new table
    let prev[0..k, V] be a new table
    for each v in V
        P[0][v] = 0
    P[0][v0] = 1

    // Process each sound
    for i = 1 to k
        for each v in V
            P[i][v] = 0
        for each u in V
            if P[i-1][u] > 0
                for each edge (u, v) in Adj[u]
                    if sigma(u, v) == s[i]
                        newprob = P[i-1][u] * p(u, v)
                        if newprob > P[i][v]
                            P[i][v] = newprob
                            prev[i][v] = u

    // Find best ending vertex
    best = 0
    best-vertex = NIL
    for each v in V
        if P[k][v] > best
            best = P[k][v]
            best-vertex = v

    if best-vertex == NIL
        return NO-SUCH-PATH

    // Reconstruct path
    path = reconstruct from prev table (as in part a)
    return path with probability best
{% endcapture %}

**Running Time:** For each of the $$k$$ sounds, we examine each edge once. Total time is $$O(k \cdot E)$$.

**Space:** $$O(k \cdot V)$$ for the probability and predecessor tables. We can reduce to $$O(V)$$ for just the probability computation (keeping only two rows), but reconstructing the path requires the full $$O(k \cdot V)$$ space.

{% capture note %}
The Viterbi algorithm is the backbone of Hidden Markov Model (HMM) decoding. In speech recognition, the vertices represent phoneme states, edges represent transitions between phonemes, and the sounds are the acoustic observations. The algorithm efficiently finds the most likely sequence of phonemes given the audio input.
