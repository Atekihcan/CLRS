> ***Longest simple path in a directed acyclic graph***
>
> Suppose that we are given a directed acyclic graph $$G = (V, E)$$ with real-valued edge weights and two distinguished vertices $$s$$ and $$t$$. Describe a dynamic-programming approach for finding a longest weighted simple path from $$s$$ to $$t$$. What does the subproblem graph look like? What is the efficiency of your algorithm?

Finding the longest path in a general graph is NP-hard, but the key insight here is that we're working with a DAG (directed acyclic graph). In a DAG, we can process vertices in a specific order (topological order) that guarantees we've solved all "prerequisite" subproblems before tackling the current one. This is what makes dynamic programming applicable.

Think of it like this: in a project with dependencies, you can always find an ordering where each task comes after all its prerequisites. Similarly, in a DAG, topological sorting gives us an ordering where for every edge $$(u, v)$$, vertex $$u$$ comes before $$v$$.

### Characterizing the Optimal Substructure

If we have a longest path from $$s$$ to $$t$$ that passes through some intermediate vertex $$v$$, then the subpath from $$s$$ to $$v$$ must itself be a longest path from $$s$$ to $$v$$. If it weren't, we could substitute a longer subpath and get a longer total path, contradicting our assumption. This optimal substructure property is the hallmark of dynamic programming problems.

### The Dynamic Programming Solution

Let $$\text{dist}[v]$$ denote the length of the longest path from $$s$$ to $$v$$. The recurrence relation is:

$$\text{dist}[v] = \max_{(u, v) \in E} \{ \text{dist}[u] + w(u, v) \}$$

where $$w(u, v)$$ is the weight of edge $$(u, v)$$. The base case is $$\text{dist}[s] = 0$$.

The algorithm proceeds as follows:

1. Perform a topological sort on $$G$$
2. Initialize $$\text{dist}[s] = 0$$ and $$\text{dist}[v] = -\infty$$ for all other vertices
3. Process vertices in topological order; for each vertex $$u$$, update $$\text{dist}[v]$$ for all edges $$(u, v)$$
4. Return $$\text{dist}[t]$$

{% capture code %}
Longest-Path-DAG(G, s, t, w)
    topologically sort the vertices of G
    let dist[1..V] be a new array
    dist[s] = 0
    for each vertex v != s in topological order
        dist[v] = -infinity
    for each vertex u in topological order
        for each edge (u, v) in Adj[u]
            if dist[v] < dist[u] + w(u, v)
                dist[v] = dist[u] + w(u, v)
    return dist[t]

### The Subproblem Graph

The subproblem graph looks exactly like the original input graph $$G$$ itself. Each vertex in the subproblem graph represents the problem "find the longest path from $$s$$ to this vertex." There is an edge from subproblem $$u$$ to subproblem $$v$$ if and only if there is an edge $$(u, v)$$ in the original graph, because to solve the longest path to $$v$$, we need to know the longest paths to all vertices $$u$$ that have edges pointing to $$v$$.

This is a beautiful example of a problem where the subproblem graph has a direct correspondence with the input structure. Since $$G$$ is a DAG, the subproblem graph is also a DAG, which means there are no circular dependencies among subproblems.

### Efficiency Analysis

The topological sort takes $$\Theta(V + E)$$ time using depth-first search. The main loop processes each vertex once and examines each edge once, also taking $$\Theta(V + E)$$ time. The space requirement is $$\Theta(V)$$ for the distance array.

Therefore, the total running time is $$\Theta(V + E)$$, which is optimal since we need to examine every vertex and edge at least once to solve the problem. This linear time complexity demonstrates the power of dynamic programming on DAGs, in contrast with the NP-hard longest path problem on general graphs with cycles.
