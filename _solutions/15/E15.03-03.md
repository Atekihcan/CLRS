> Consider a variant of the matrix-chain multiplication problem in which the goal is to parenthesize the sequence of matrices so as to maximize, rather than minimize, the number of scalar multiplications. Does this problem exhibit optimal substructure?

Yes, this maximization variant exhibits optimal substructure, and the reasoning is almost identical to the minimization version.

Consider an optimal parenthesization that maximizes the number of scalar multiplications for the product $$A_i A_{i+1} \cdots A_j$$. This parenthesization must split the product at some position $$k$$, resulting in $$A_i \cdots A_k$$ multiplied by $$A_{k+1} \cdots A_j$$. The total cost is the cost of computing $$A_i \cdots A_k$$, plus the cost of computing $$A_{k+1} \cdots A_j$$, plus the cost $$p_{i-1} p_k p_j$$ of multiplying the two resulting matrices.

Now, suppose the parenthesization of $$A_i \cdots A_k$$ within our optimal solution is not itself a maximum-cost parenthesization. Then there exists some other parenthesization of $$A_i \cdots A_k$$ that requires more scalar multiplications. If we substitute this higher-cost parenthesization into our solution, we obtain a parenthesization for the entire product $$A_i \cdots A_j$$ with strictly more multiplications. This contradicts our assumption that we started with an optimal (maximum-cost) solution.

By the same cut-and-paste argument, the parenthesization of $$A_{k+1} \cdots A_j$$ must also be optimal for that subchain.

The recurrence for the maximum cost $$m[i, j]$$ is:

$$m[i, j] = \begin{cases}
0 & \text{if } i = j \\
\max_{i \le k < j} \{m[i, k] + m[k+1, j] + p_{i-1} p_k p_j\} & \text{if } i < j
\end{cases}$$

This is identical to the standard matrix-chain recurrence, except we use $$\max$$ instead of $$\min$$. We can solve this with the same dynamic programming approach, filling in the table bottom-up and taking $$O(n^3)$$ time.

{% capture note %}
The optimal substructure property depends on the structure of how subproblems combine, not on whether we minimize or maximize. Both versions share the same subproblem decomposition: splitting the chain at position $$k$$. The only difference is the choice of $$k$$, and both versions require optimal solutions to subproblems regardless of the optimization direction.
{% endcapture %}
