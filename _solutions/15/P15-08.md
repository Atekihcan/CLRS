> ***Image compression by seam carving***
>
> We are given a color picture consisting of an $$m \times n$$ array $$A[1..m, 1..n]$$ of pixels, where each pixel specifies a triple of red, green, and blue (RGB) intensities. Suppose that we wish to compress this picture slightly. Specifically, we wish to remove one pixel from each of the $$m$$ rows, so that the whole picture becomes one pixel narrower. To avoid disturbing visual effects, however, we require that the pixels removed in two adjacent rows be in the same or adjacent columns; the pixels removed form a "seam" from the top row to the bottom row where successive pixels in the seam are adjacent vertically or diagonally.
>
> 1. Show that the number of such possible seams grows at least exponentially in $$m$$, assuming that $$n > 1$$.
>
> 2. Suppose now that along with each pixel $$A[i, j]$$, we have calculated a real-valued disruption measure $$d[i, j]$$, indicating how disruptive it would be to remove pixel $$A[i, j]$$. Intuitively, the lower a pixel's disruption measure, the more similar the pixel is to its neighbors. Suppose further that we define the disruption measure of a seam to be the sum of the disruption measures of its pixels. Give an algorithm to find a seam with the lowest disruption measure. How efficient is your algorithm?

This problem describes [seam carving](https://en.wikipedia.org/wiki/Seam_carving){:target="_blank"}, a content-aware image resizing technique. Rather than uniformly scaling an image, seam carving removes (or adds) "seams" of low importance, preserving important features like faces while eliminating less important regions like sky or grass.

### A. Exponential Number of Seams

A seam is a connected path from the top row to the bottom row, where each step moves down one row and can move left, stay in the same column, or move right. This gives us (roughly) 3 choices at each row.

Starting from any column in row 1, at each subsequent row we have at least 2 choices (we can always go straight down or to at least one diagonal). At the edges we have 2 choices; in the middle we have 3.

Even with the most conservative estimate (2 choices per row), the number of seams is at least $$2^{m-1}$$, which grows exponentially in $$m$$.

More precisely, for the middle columns we have 3 choices, so the number of seams is approximately $$n \cdot 3^{m-1}$$, which is $$\Omega(3^m)$$. This is definitely exponential in $$m$$.

{% capture note %}
The exact count is more complex because edge columns have only 2 choices (can't go outside the image). But even the lower bound of $$2^{m-1}$$ proves exponential growth, making brute-force enumeration infeasible for images of any reasonable size.
{% endcapture %}

### B. Finding the Minimum Disruption Seam

This is a classic dynamic programming problem. We build up the minimum disruption to reach each pixel from the top.

Let $$D[i][j]$$ be the minimum total disruption of any seam ending at pixel $$(i, j)$$.

**Base case (first row):**

$$D[1][j] = d[1][j]$$ for all $$j$$

**Recurrence (rows 2 to m):**

$$D[i][j] = d[i][j] + \min(D[i-1][j-1], D[i-1][j], D[i-1][j+1])$$

We take the minimum of the three pixels in the row above that can connect to position $$(i, j)$$. At the boundaries, we only consider the valid neighbors:
- At $$j = 1$$: only consider $$D[i-1][1]$$ and $$D[i-1][2]$$
- At $$j = n$$: only consider $$D[i-1][n-1]$$ and $$D[i-1][n]$$

**Finding the optimal seam:**

After filling the table, the minimum value in the last row $$D[m][j]$$ gives the optimal seam's disruption. We trace back from there to reconstruct the seam.

{% capture code %}
Find-Min-Seam(d, m, n)
    let D[1..m, 1..n] be a new table
    let parent[1..m, 1..n] be a new table

    // Base case: first row
    for j = 1 to n
        D[1][j] = d[1][j]
        parent[1][j] = 0

    // Fill table row by row
    for i = 2 to m
        for j = 1 to n
            // Find minimum among valid predecessors
            best = D[i-1][j]
            best-col = j
            if j > 1 and D[i-1][j-1] < best
                best = D[i-1][j-1]
                best-col = j - 1
            if j < n and D[i-1][j+1] < best
                best = D[i-1][j+1]
                best-col = j + 1

            D[i][j] = d[i][j] + best
            parent[i][j] = best-col

    // Find best ending column
    min-disruption = infinity
    end-col = 0
    for j = 1 to n
        if D[m][j] < min-disruption
            min-disruption = D[m][j]
            end-col = j

    // Reconstruct the seam
    let seam[1..m] be a new array
    seam[m] = end-col
    for i = m downto 2
        seam[i-1] = parent[i][seam[i]]

    return seam, min-disruption
{% endcapture %}

### Efficiency Analysis

**Time complexity:** We have $$m \times n$$ table entries, and each entry requires $$O(1)$$ time to compute (just comparing 3 values). Finding the minimum in the last row takes $$O(n)$$. Reconstructing the seam takes $$O(m)$$. Total time: $$\Theta(mn)$$.

**Space complexity:** The table requires $$\Theta(mn)$$ space. However, we can reduce this to $$\Theta(n)$$ for just computing the minimum disruption value (keeping only two rows). To reconstruct the seam with $$\Theta(n)$$ space, we would need to use a different technique (like storing only parent pointers and recomputing as needed, or using the divide-and-conquer approach from Hirschberg's algorithm for LCS).

{% capture note %}
In practice, seam carving is applied repeatedly to reduce image width by multiple pixels. Each removal requires recomputing the disruption values for affected pixels. The original paper by Avidan and Shamir (SIGGRAPH 2007) shows beautiful examples of content-aware resizing that would be impossible with traditional scaling.
{% endcapture %}
