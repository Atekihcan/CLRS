> ***Coin changing***
>
> Consider the problem of making change for $$n$$ cents using the fewest number of coins. Assume that each coin's value is an integer.
>
> 1. Describe a greedy algorithm to make change consisting of quarters, dimes, nickels, and pennies. Prove that your algorithm yields an optimal solution.
> 2. Suppose that the available coins are in the denominations that are powers of $$c$$, i.e., the denominations are $$c^0, c^1, \ldots, c^k$$ for some integers $$c > 1$$ and $$k \ge 1$$. Show that the greedy algorithm always yields an optimal solution.
> 3. Give a set of coin denominations for which the greedy algorithm does not yield an optimal solution. Your set should include a penny so that there is a solution for every value of $$n$$.
> 4. Give an $$O(nk)$$-time algorithm that makes change for any set of $$k$$ different coin denominations, assuming that one of the coins is a penny.

The coin changing problem is a classic example that illustrates when greedy algorithms work and when they fail. The key insight is that the greedy approach (always picking the largest coin that fits) works only when the coin denominations have a special structure.

### A

For US currency with quarters (25 cents), dimes (10 cents), nickels (5 cents), and pennies (1 cent), the greedy algorithm repeatedly selects the largest denomination coin that does not exceed the remaining amount.

{% capture code %}
Make-Change(n)
    coins = 0
    for each denomination d in {25, 10, 5, 1} (decreasing order)
        coins = coins + floor(n / d)
        n = n mod d
    return coins
{% endcapture %}

To prove optimality, we need to show that any optimal solution uses at most a certain number of each coin type. Consider an optimal solution and observe the following constraints that any optimal solution must satisfy:

- At most 4 pennies (5 pennies = 1 nickel)
- At most 1 nickel (2 nickels = 1 dime)
- At most 2 dimes (3 dimes = 30 cents, better as quarter + nickel with same count but we need to be careful)

More precisely, an optimal solution can have at most 4 pennies (since 5 pennies can be replaced by 1 nickel), and the value from nickels and pennies combined is at most 9 cents (otherwise 2 nickels = 1 dime, or 1 nickel + 5 pennies = 1 dime). Similarly, the combined value from dimes, nickels, and pennies is at most 24 cents (otherwise we could use a quarter).

This means that once we decide how many quarters to use, the remaining amount (which is at most 24 cents) has a unique optimal way to make change. The greedy algorithm naturally picks the maximum number of quarters first, then fills in optimally for the remainder. Since the greedy choice of taking as many quarters as possible is always safe (any solution not using the maximum quarters would have more total coins to cover the same amount with smaller denominations), the algorithm produces an optimal solution.

### B

When denominations are powers of $$c$$ (i.e., $$1, c, c^2, \ldots, c^k$$), the greedy algorithm always works. The key observation is that $$c$$ coins of denomination $$c^i$$ can always be replaced by a single coin of denomination $$c^{i+1}$$.

We prove optimality by showing that any optimal solution uses fewer than $$c$$ coins of each denomination except possibly the largest. Suppose an optimal solution uses $$c$$ or more coins of denomination $$c^i$$ for some $$i < k$$. Then we could replace $$c$$ of these coins with a single coin of denomination $$c^{i+1}$$, reducing the total number of coins. This contradicts optimality.

Now consider the greedy algorithm. When we represent $$n$$ in base $$c$$, we get $$n = a_k c^k + a_{k-1} c^{k-1} + \cdots + a_1 c + a_0$$, where $$0 \le a_i < c$$ for each $$i$$. The greedy algorithm effectively computes this representation: it uses $$\lfloor n / c^k \rfloor$$ coins of denomination $$c^k$$, then $$\lfloor (n \mod c^k) / c^{k-1} \rfloor$$ coins of denomination $$c^{k-1}$$, and so on.

The total number of coins is $$\sum_{i=0}^{k} a_i$$. Since any optimal solution must use fewer than $$c$$ coins of each denomination, and the greedy solution uses exactly the digits in the base-$$c$$ representation (which are all less than $$c$$), the greedy solution is optimal. In fact, this is the unique optimal solution.

### C

A simple counterexample uses denominations $$\{1, 3, 4\}$$. Consider making change for $$n = 6$$ cents:

- **Greedy solution**: Take a 4-cent coin first, then two 1-cent coins, for a total of 3 coins.
- **Optimal solution**: Take two 3-cent coins, for a total of 2 coins.

The greedy algorithm fails here because the denominations don't have the "covering" property of powers of $$c$$. Another classic counterexample is $$\{1, 6, 10\}$$ for $$n = 12$$: greedy gives $$10 + 1 + 1$$ (3 coins) while optimal is $$6 + 6$$ (2 coins).

### D

We use dynamic programming to solve the general coin changing problem. Let $$c[j]$$ be the minimum number of coins needed to make change for $$j$$ cents, given denominations $$d_1, d_2, \ldots, d_k$$ (where $$d_1 = 1$$).

The recurrence is:

$$c[j] = \begin{cases} 0 & \text{if } j = 0 \\ \min_{1 \le i \le k, d_i \le j} \{c[j - d_i] + 1\} & \text{if } j > 0 \end{cases}$$

The idea is straightforward: to make change for $$j$$ cents, we try each coin denomination $$d_i$$ and see what's the minimum if we use that coin. We take the best option across all valid coin choices.

{% capture code %}
Coin-Change(n, d, k)
    let c[0..n] be a new array
    c[0] = 0
    for j = 1 to n
        c[j] = infinity
        for i = 1 to k
            if d[i] <= j and c[j - d[i]] + 1 < c[j]
                c[j] = c[j - d[i]] + 1
    return c[n]
{% endcapture %}
{% include clrs_code.html title="Coin-Change(n, d, k)" %}

The outer loop runs $$n$$ times and the inner loop runs $$k$$ times, giving $$O(nk)$$ time complexity. The space complexity is $$O(n)$$ for the array $$c$$. Since one of the coins is a penny (value 1), we're guaranteed to find a solution for every positive $$n$$.

To reconstruct which coins to use, we can maintain a second array that tracks which coin was chosen at each step, then trace back from $$c[n]$$ to $$c[0]$$.
