> Using Figure 10.2 as a model, illustrate the result of each operation in the sequence $$\textsc{Enqueue}$$($$Q$$, 4), $$\textsc{Enqueue}$$($$Q$$, 1), $$\textsc{Enqueue}$$($$Q$$, 3), $$\textsc{Dequeue}$$($$Q$$), $$\textsc{Enqueue}$$($$Q$$, 8), and $$\textsc{Dequeue}$$($$Q$$) on an initially empty queue $$Q$$ stored in array $$Q[1..6]$$.

We start with an empty queue where $$Q.\textit{head} = Q.\textit{tail} = 1$$. The queue uses a circular array, so when indices reach the end, they wrap around to the beginning.

**After $$\textsc{Enqueue}$$($$Q$$, 4):** The first element is inserted at the tail position. We place 4 at $$Q[1]$$, then increment tail to 2.

```
[4][_][_][_][_][_]
 ↑  ↑
head tail
 1   2
```

**After $$\textsc{Enqueue}$$($$Q$$, 1):** We insert 1 at position 2, then increment tail to 3.

```
[4][1][_][_][_][_]
 ↑     ↑
head  tail
 1     3
```

**After $$\textsc{Enqueue}$$($$Q$$, 3):** We insert 3 at position 3, then increment tail to 4.

```
[4][1][3][_][_][_]
 ↑        ↑
head     tail
 1        4
```

**After $$\textsc{Dequeue}$$($$Q$$):** We remove the element at the head (4). The head pointer advances to 2, returning the value 4.

```
[4][1][3][_][_][_]
    ↑     ↑
   head  tail
    2     4
```

**After $$\textsc{Enqueue}$$($$Q$$, 8):** We insert 8 at position 4, then increment tail to 5.

```
[4][1][3][8][_][_]
    ↑        ↑
   head     tail
    2        5
```

**After $$\textsc{Dequeue}$$($$Q$$):** We remove the element at the head (1). The head pointer advances to 3, returning the value 1.

```
[4][1][3][8][_][_]
       ↑     ↑
      head  tail
       3     5
```

The final state has $$Q.\textit{head} = 3$$ and $$Q.\textit{tail} = 5$$, with elements 3 and 8 in the queue (at positions 3 and 4 respectively).
