> Suppose that in a 0-1 knapsack problem, the order of the items when sorted by increasing weight is the same as their order when sorted by decreasing value. Give an efficient algorithm to find an optimal solution to this variant of the knapsack problem, and argue that your algorithm is correct.

The constraint tells us that lighter items are more valuable. If we sort items by increasing weight, we get $$w_1 \leq w_2 \leq \cdots \leq w_n$$, and simultaneously $$v_1 \geq v_2 \geq \cdots \geq v_n$$. This means item 1 is both the lightest and the most valuable, item 2 is the second lightest and second most valuable, and so on.

This special structure makes a greedy approach optimal: take items in order of increasing weight (equivalently, decreasing value) until the knapsack is full.

{% capture code %}
Special-Knapsack(v, w, n, W)
    // Assume items already sorted by increasing weight (decreasing value)
    total_weight = 0
    total_value = 0
    S = empty set
    for i = 1 to n
        if total_weight + w[i] ≤ W
            S = S ∪ {i}
            total_weight = total_weight + w[i]
            total_value = total_value + v[i]
    return S
{% endcapture %}

**Correctness Proof:** We prove the greedy-choice property using an exchange argument.

Let $$G$$ be the greedy solution that selects items $$\{1, 2, \ldots, k\}$$ (the first $$k$$ items until the knapsack is full). Suppose $$O$$ is an optimal solution different from $$G$$. Let $$j$$ be the smallest index where $$G$$ and $$O$$ differ.

**Case 1:** $$j \in G$$ but $$j \notin O$$. Since $$j$$ was chosen by greedy but not by optimal, $$O$$ must contain some item $$\ell > j$$ (otherwise $$O$$ could add $$j$$ and improve). Create $$O' = (O \setminus \{\ell\}) \cup \{j\}$$. Since $$w_j \leq w_\ell$$ and $$v_j \geq v_\ell$$ (by our sorting), $$O'$$ fits in the knapsack and has value at least as good as $$O$$. This contradicts $$O$$ being strictly better than a solution containing $$j$$.

**Case 2:** $$j \notin G$$ but $$j \in O$$. This means greedy skipped item $$j$$, which only happens if adding $$j$$ would exceed capacity $$W$$. But greedy already took items $$1, \ldots, j-1$$ (all lighter and more valuable than $$j$$). If $$O$$ includes $$j$$, it cannot include all of $$1, \ldots, j-1$$ while fitting $$j$$. Any item $$i < j$$ that $$O$$ excludes could replace $$j$$ with better or equal value, contradicting optimality of $$O$$.

By repeated exchanges, we can transform $$O$$ into $$G$$ without losing value, proving $$G$$ is optimal.

**Running time:** $$O(n \log n)$$ for sorting (if not pre-sorted), plus $$O(n)$$ for the greedy selection. Total: $$O(n \log n)$$.
