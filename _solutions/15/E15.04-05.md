> Give an $$O(n^2)$$-time algorithm to find the longest monotonically increasing subsequence of a sequence of $$n$$ numbers.

The longest monotonically increasing subsequence (LIS) problem asks us to find the longest subsequence of a given sequence where each element is strictly greater than the previous one. For example, in $$\langle 10, 22, 9, 33, 21, 50, 41, 60, 80 \rangle$$, one LIS is $$\langle 10, 22, 33, 50, 60, 80 \rangle$$ with length 6.

One elegant approach is to reduce LIS to LCS. If we sort the original sequence to get a new sequence, then the LCS of the original and sorted sequences gives us the LIS. However, this requires $$O(n \log n)$$ for sorting plus $$O(n^2)$$ for LCS, and uses $$O(n^2)$$ space for the LCS table.

A more direct dynamic programming approach works as follows. Let $$L[i]$$ be the length of the longest increasing subsequence that **ends at** index $$i$$. For each position $$i$$, we look at all previous positions $$j < i$$ where $$A[j] < A[i]$$. If we can extend the LIS ending at $$j$$ by appending $$A[i]$$, we consider this as a candidate.

The recurrence is:

$$L[i] = 1 + \max_{j < i, A[j] < A[i]} L[j]$$

If no valid $$j$$ exists (i.e., $$A[i]$$ is smaller than all previous elements), then $$L[i] = 1$$ (the subsequence containing just $$A[i]$$).

{% capture code %}
LIS-Length(A)
    n = A.length
    let L[1..n] be a new array
    for i = 1 to n
        L[i] = 1
        for j = 1 to i - 1
            if A[j] < A[i] and L[j] + 1 > L[i]
                L[i] = L[j] + 1
    return max(L[1], L[2], ..., L[n])
{% endcapture %}

To reconstruct the actual subsequence, we keep track of predecessors:

{% capture code %}
LIS(A)
    n = A.length
    let L[1..n] and P[1..n] be new arrays
    for i = 1 to n
        L[i] = 1
        P[i] = NIL
        for j = 1 to i - 1
            if A[j] < A[i] and L[j] + 1 > L[i]
                L[i] = L[j] + 1
                P[i] = j
    // Find the index with maximum L value
    max_idx = 1
    for i = 2 to n
        if L[i] > L[max_idx]
            max_idx = i
    // Reconstruct the subsequence
    Print-LIS(A, P, max_idx)
{% endcapture %}

The running time is $$O(n^2)$$ because:
- The outer loop runs $$n$$ times
- For each $$i$$, the inner loop runs $$i - 1$$ times
- Total iterations: $$\sum_{i=1}^{n}(i-1) = \frac{n(n-1)}{2} = O(n^2)$$

The space complexity is $$O(n)$$ for the $$L$$ and $$P$$ arrays.

{% capture note %}
The LIS problem has deep connections to other problems in computer science. It's related to patience sorting (a card game that naturally computes LIS), Young tableaux, and the Robinson-Schensted correspondence in combinatorics. The $$O(n^2)$$ solution we've shown here is simple and intuitive, but Exercise 15.4-6 asks for an $$O(n \log n)$$ solution using a clever binary search technique.
