> Consider an ordinary binary min-heap data structure with $$n$$ elements supporting the instructions $$\textsc{Insert}$$ and $$\textsc{Extract-Min}$$ in $$O(\lg n)$$ worst-case time. Give a potential function $$\Phi$$ such that the amortized cost of $$\textsc{Insert}$$ is $$O(\lg n)$$ and the amortized cost of $$\textsc{Extract-Min}$$ is $$O(1)$$, and show that it works.

This exercise asks us to find a potential function that makes $$\textsc{Extract-Min}$$ appear cheap in an amortized sense, even though it actually costs $$O(\lg n)$$ in the worst case. The idea is that each $$\textsc{Insert}$$ operation should "prepay" for the cost of a future extraction by storing potential energy in the heap.

Think of it like a deposit system: every time you add an item to the heap, you put some money in a jar. When you later extract the minimum, you use the money from the jar to pay for the work of restructuring the heap.

Define the potential function as:

$$\Phi(H) = c \cdot n \cdot \lg(n + 1)$$

where $$n$$ is the number of elements currently in the heap and $$c$$ is a sufficiently large constant (we will determine its value shortly).

**Analyzing $$\textsc{Insert}$$:**

When we insert into a heap of size $$n$$, the heap grows to size $$n + 1$$. The actual cost is $$O(\lg n)$$ because the new element may bubble up through at most $$\lg n$$ levels.

$$\begin{align*}
\hat{c}_{\textsc{Insert}} &= c_{\textsc{Insert}} + \Phi(H_{\text{after}}) - \Phi(H_{\text{before}}) \\
                          &= O(\lg n) + c(n+1)\lg(n+2) - cn\lg(n+1) \\
                          &= O(\lg n) + c \left[ (n+1)\lg(n+2) - n\lg(n+1) \right]
\end{align*}$$

The change in potential is positive, adding to the amortized cost. Using the approximation that $$(n+1)\lg(n+2) - n\lg(n+1) = O(\lg n)$$, we get:

$$\hat{c}_{\textsc{Insert}} = O(\lg n)$$

**Analyzing $$\textsc{Extract-Min}$$:**

When we extract the minimum from a heap of size $$n$$, the heap shrinks to size $$n - 1$$. The actual cost is $$O(\lg n)$$ because after swapping the last element to the root, it may need to bubble down through at most $$\lg n$$ levels.

$$\begin{align*}
\hat{c}_{\textsc{Extract-Min}} &= c_{\textsc{Extract-Min}} + \Phi(H_{\text{after}}) - \Phi(H_{\text{before}}) \\
                               &= O(\lg n) + c(n-1)\lg n - cn\lg(n+1) \\
                               &= O(\lg n) - c \left[ n\lg(n+1) - (n-1)\lg n \right]
\end{align*}$$

For large enough $$c$$, the decrease in potential (which is $$\Omega(\lg n)$$) pays for the actual cost of the extraction. Specifically, $$n\lg(n+1) - (n-1)\lg n \ge \lg n$$ for $$n \ge 1$$, so choosing $$c$$ large enough ensures:

$$\hat{c}_{\textsc{Extract-Min}} = O(\lg n) - \Omega(\lg n) = O(1)$$

The potential function is always nonnegative since $$n \ge 0$$ and $$\lg(n+1) \ge 0$$ for $$n \ge 0$$. Starting with an empty heap where $$\Phi(H_0) = 0$$, we have $$\Phi(H_i) \ge 0$$ for all $$i$$, so the total amortized cost bounds the total actual cost.
