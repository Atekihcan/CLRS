> ***Bitonic euclidean traveling-salesman problem***
>
> In the euclidean traveling-salesman problem, we are given a set of $$n$$ points in the plane, and we wish to find the shortest closed tour that connects all $$n$$ points. The general problem is NP-hard, and its solution is therefore believed to require more than polynomial time (see Chapter 34).
>
> J. L. Bentley has suggested that we simplify the problem by restricting our attention to bitonic tours, that is, tours that start at the leftmost point, go strictly rightward to the rightmost point, and then go strictly leftward back to the starting point.
>
> Describe an $$O(n^2)$$-time algorithm for determining an optimal bitonic tour. You may assume that no two points have the same x-coordinate and that all operations on real numbers take unit time. (Hint: Scan left to right, maintaining optimal possibilities for the two parts of the tour.)

A bitonic tour has a very specific structure: imagine traveling from the leftmost point, always moving right until you reach the rightmost point, then always moving left until you return to the start. The tour looks like two paths that together visit every point exactly once.

The key insight is that at any point during our scan from left to right, we can describe the state of a partial tour using just two pieces of information: which point each of the two "chains" (the upper and lower parts of the tour) has reached so far.

### Setting Up the Subproblems

First, sort all $$n$$ points by their x-coordinates. Let the sorted points be $$p_1, p_2, \ldots, p_n$$, where $$p_1$$ is the leftmost and $$p_n$$ is the rightmost.

Let $$d(i, j)$$ denote the Euclidean distance between points $$p_i$$ and $$p_j$$. We define $$B[i][j]$$ to be the length of the shortest bitonic path that:
- Starts at $$p_i$$, goes strictly leftward to $$p_1$$
- Starts at $$p_j$$, goes strictly leftward to $$p_1$$
- Together, these two paths visit all points $$p_1, p_2, \ldots, p_{\max(i,j)}$$

By symmetry, $$B[i][j] = B[j][i]$$, so we only need to consider cases where $$i \geq j$$.

### The Recurrence

For the base case, $$B[1][1] = 0$$ (both chains start at the same point).

When we add point $$p_i$$ (where $$i > j$$), it must connect to one of the two chains. If it extends the chain currently ending at $$p_{i-1}$$:

$$B[i][j] = B[i-1][j] + d(i-1, i) \quad \text{when } i > j + 1$$

If point $$p_i$$ switches to extend the other chain (currently at $$p_j$$), the previous point $$p_{i-1}$$ must be the endpoint of the first chain:

$$B[i][i-1] = \min_{1 \leq k < i-1} \{ B[i-1][k] + d(k, i) \}$$

The final tour must close the loop by connecting $$p_n$$ to both chains. Since both chains must meet at $$p_n$$:

$$\text{Tour length} = \min_{1 \leq j < n} \{ B[n][j] + d(j, n) \}$$

Wait, this isn't quite right. Let's reconsider.

### Refined Formulation

A cleaner approach: let $$B[i][j]$$ (where $$i > j$$) be the minimum total length of two paths starting at $$p_i$$ and $$p_j$$ respectively, going leftward, covering all points from $$p_1$$ to $$p_i$$, and both ending at $$p_1$$.

**Base case:** $$B[2][1] = d(1, 2)$$

**Recurrence for $$i > j + 1$$:** The new point $$p_i$$ must extend the path that previously ended at $$p_{i-1}$$:

$$B[i][j] = B[i-1][j] + d(i-1, i)$$

**Recurrence for $$j = i - 1$$:** Point $$p_i$$ connects to some earlier point $$p_k$$ on the other path:

$$B[i][i-1] = \min_{1 \leq k < i-1} \{ B[i-1][k] + d(k, i) \}$$

**Final answer:** Both paths must close at $$p_n$$:

$$\text{Optimal tour} = \min_{1 \leq j < n} \{ B[n][j] + d(j, n) \}$$

### Running Time Analysis

Sorting the points takes $$O(n \log n)$$. Computing all pairwise distances takes $$O(n^2)$$. The DP table has $$O(n^2)$$ entries, and most entries take $$O(1)$$ time to compute since we only add one distance. However, computing $$B[i][i-1]$$ requires minimizing over $$O(i)$$ previous values, which sums to $$O(n^2)$$ across all $$i$$.

Therefore, the total running time is $$O(n^2)$$.
