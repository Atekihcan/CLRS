> Describe an efficient algorithm that, given a set $$\{x_1, x_2, \ldots, x_n\}$$ of points on the real line, determines the smallest set of unit-length closed intervals that contains all of the given points. Argue that your algorithm is correct.

Think of placing rulers (unit-length intervals) on a number line to cover scattered points. The greedy insight is: always place the next ruler starting at the leftmost uncovered point. This ensures we cover that point while reaching as far right as possible, maximizing the chance of covering other points.

{% capture code %}
Minimum-Intervals(X)
    sort X in increasing order
    intervals = empty list
    i = 1
    while i ≤ n
        // Start new interval at leftmost uncovered point
        left = X[i]
        right = left + 1
        intervals.append([left, right])
        // Skip all points covered by this interval
        while i ≤ n and X[i] ≤ right
            i = i + 1
    return intervals
{% endcapture %}

**Correctness Proof:** We prove the greedy-choice property using an exchange argument.

Let $$p$$ be the leftmost point, and let $$G = [p, p+1]$$ be the greedy choice (an interval starting at $$p$$). We claim some optimal solution includes an interval that can be replaced by $$G$$ without increasing the count.

Consider any optimal solution $$O$$. Some interval $$I = [a, a+1] \in O$$ must cover point $$p$$, so $$a \leq p \leq a+1$$, which means $$a \leq p$$.

Construct $$O'$$ by replacing $$I$$ with $$G$$:
- $$G$$ covers $$p$$ (since $$p \in [p, p+1]$$).
- Any point $$q$$ covered by $$I$$ satisfies $$a \leq q \leq a+1$$. If $$q < p$$, this contradicts $$p$$ being leftmost. So $$q \geq p$$, meaning $$q \in [p, p+1] = G$$ as long as $$q \leq p+1$$. Since $$q \leq a+1$$ and $$a \leq p$$, we have $$q \leq p+1$$.

Thus every point covered by $$I$$ is also covered by $$G$$, so $$O'$$ is valid with the same number of intervals.

**Optimal Substructure:** After placing interval $$G = [p, p+1]$$ at the leftmost point $$p$$, all points in $$[p, p+1]$$ are covered. The remaining problem (covering points greater than $$p+1$$) is a smaller instance of the same problem. An optimal solution combines the greedy choice with an optimal solution to this subproblem.

**Running Time:** Sorting takes $$O(n \log n)$$. The main loop processes each point exactly once, taking $$O(n)$$ time. Total: $$O(n \log n)$$.

{% capture note %}
Starting each interval at a point (rather than before it) is crucial. If we allowed intervals to start before the leftmost point, we might cover the same points with a longer reach, but this provides no benefit since points to the left of the leftmost point don't exist.
{% endcapture %}
