> ***Longest palindrome subsequence***
>
> A palindrome is a nonempty string over some alphabet that reads the same forward and backward. Examples of palindromes are all strings of length 1, civic, racecar, and aibohphobia (fear of palindromes).
>
> Give an efficient algorithm to find the longest palindrome that is a subsequence of a given input string. For example, given the input character, your algorithm should return carac. What is the running time of your algorithm?

A palindrome reads the same forwards and backwards. Think of words like "racecar" or "civic". The key insight for this problem is recognizing a beautiful connection: a palindromic subsequence of a string $$S$$ is essentially a common subsequence of $$S$$ and its reverse $$S^R$$.

For example, consider the string "character". Its reverse is "retcarahc". The longest common subsequence of these two strings is "carac", which is indeed a palindrome!

This reduction is elegant because:
1. Any common subsequence of $$S$$ and $$S^R$$ must be a palindrome (since it appears in both forward and reverse readings)
2. Any palindromic subsequence of $$S$$ is certainly a subsequence of $$S^R$$ as well (reading it backwards gives the same sequence)

### Algorithm

We can directly apply the Longest Common Subsequence (LCS) algorithm from Section 15.4:

{% capture code %}
Longest-Palindrome-Subsequence(X)
    n = X.length
    let Y[1..n] be a new array
    for i = 1 to n
        Y[i] = X[n - i + 1]
    return LCS-Length(X, Y)
{% endcapture %}
{% include clrs_code.html title="Longest-Palindrome-Subsequence(X)" %}

Alternatively, we can develop a direct dynamic programming formulation. Let $$L[i][j]$$ be the length of the longest palindromic subsequence in the substring $$X[i..j]$$. The recurrence is:

$$L[i][j] = \begin{cases}
1 & \text{if } i = j \\
2 & \text{if } j = i + 1 \text{ and } X[i] = X[j] \\
1 & \text{if } j = i + 1 \text{ and } X[i] \neq X[j] \\
L[i+1][j-1] + 2 & \text{if } j > i + 1 \text{ and } X[i] = X[j] \\
\max(L[i+1][j], L[i][j-1]) & \text{if } j > i + 1 \text{ and } X[i] \neq X[j]
\end{cases}$$

The intuition behind this recurrence:
- A single character is a palindrome of length 1
- For two characters, we get length 2 if they match, otherwise length 1
- If the endpoints match, they can both be part of the palindrome, so we add 2 to the best solution for the interior
- If the endpoints don't match, at least one of them can't be in our palindromic subsequence, so we take the better of excluding either endpoint

### Running Time

The LCS-based approach runs in $$\Theta(n^2)$$ time and uses $$\Theta(n^2)$$ space, where $$n$$ is the length of the input string. We compute the LCS of two strings each of length $$n$$, which requires filling in an $$n \times n$$ table.

The direct DP approach also runs in $$\Theta(n^2)$$ time. There are $$\Theta(n^2)$$ subproblems (all pairs $$(i, j)$$ where $$i \leq j$$), and each takes $$\Theta(1)$$ time to solve once its dependencies are available.

{% capture note %}
The space can be reduced to $$\Theta(n)$$ for both approaches if we only need the length (not the actual subsequence), since computing row $$i$$ only requires information from row $$i+1$$ in the direct DP formulation.
{% endcapture %}
