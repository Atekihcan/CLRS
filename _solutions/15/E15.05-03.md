> Suppose that instead of maintaining the table $$w[i, j]$$, we computed the value of $$w(i, j)$$ directly from equation (15.12) in line 9 of $$\textsc{Optimal-BST}$$ and used this computed value in line 11. How would this change affect the asymptotic running time of $$\textsc{Optimal-BST}$$?

Let's first understand what the $$w$$ table does. The value $$w(i, j)$$ represents the sum of all probabilities in the subtree containing keys $$k_i, \ldots, k_j$$:

$$w(i, j) = \sum_{l=i}^{j} p_l + \sum_{l=i-1}^{j} q_l$$

In the original $$\textsc{Optimal-BST}$$ procedure, we maintain a table $$w[i, j]$$ and compute each entry incrementally using the recurrence:

$$w[i, j] = w[i, j-1] + p_j + q_j$$

This takes $$\Theta(1)$$ time per entry because we just add two values to a previously computed sum.

Now consider what happens if we compute $$w(i, j)$$ directly from the formula each time we need it. The sum in equation (15.12) has $$(j - i + 1)$$ terms for the $$p$$ values and $$(j - i + 2)$$ terms for the $$q$$ values. So computing $$w(i, j)$$ directly takes $$\Theta(j - i)$$ time.

Let's analyze the total running time. In the $$\textsc{Optimal-BST}$$ procedure, we have three nested loops:
- The outer loop runs for $$l = 1$$ to $$n$$ (the length of the subproblem)
- The middle loop runs for $$i = 1$$ to $$n - l + 1$$
- The inner loop tries all possible roots $$r = i$$ to $$j = i + l - 1$$

For each $$(i, j)$$ pair, line 9 computes $$w[i, j]$$. If we compute this directly:

$$\begin{align*}
\text{Total time for } w \text{ computations} &= \sum_{l=1}^{n} \sum_{i=1}^{n-l+1} \Theta(j - i) \\
&= \sum_{l=1}^{n} \sum_{i=1}^{n-l+1} \Theta(l - 1) \\
&= \sum_{l=1}^{n} (n - l + 1) \cdot \Theta(l) \\
&= \Theta\left(\sum_{l=1}^{n} l(n - l + 1)\right)
\end{align*}$$

Let's evaluate this sum:

$$\begin{align*}
\sum_{l=1}^{n} l(n - l + 1) &= \sum_{l=1}^{n} (ln - l^2 + l) \\
&= n\sum_{l=1}^{n} l - \sum_{l=1}^{n} l^2 + \sum_{l=1}^{n} l \\
&= n \cdot \frac{n(n+1)}{2} - \frac{n(n+1)(2n+1)}{6} + \frac{n(n+1)}{2} \\
&= \frac{n^2(n+1)}{2} + \frac{n(n+1)}{2} - \frac{n(n+1)(2n+1)}{6} \\
&= \frac{n(n+1)}{6}\left(3n + 3 - (2n+1)\right) \\
&= \frac{n(n+1)(n+2)}{6} \\
&= \Theta(n^3)
\end{align*}$$

So just the $$w$$ computations alone take $$\Theta(n^3)$$ time.

But we must also account for the innermost loop (lines 10-14), which tries each possible root. This loop still runs $$\Theta(n^3)$$ times total across all subproblems (as in the original analysis). Each iteration does $$\Theta(1)$$ work.

Adding these together:
- $$w$$ computations: $$\Theta(n^3)$$
- Root selection loop: $$\Theta(n^3)$$

**The asymptotic running time remains $$\Theta(n^3)$$.**

{% capture note %}
While the asymptotic complexity stays the same, the constant factor roughly doubles. In the original algorithm, we do $$\Theta(1)$$ work per subproblem for the $$w$$ table, but now we do $$\Theta(l)$$ work where $$l$$ is the subproblem size. More importantly, we lose the elegant incremental computation that made the original algorithm efficient in practice. For large $$n$$, this could mean the difference between a program that runs in seconds versus minutes.
{% endcapture %}
{% include aside.html title='Why maintain the table?' %}

To summarize: eliminating the $$w$$ table does not change the asymptotic running time of $$\textsc{Optimal-BST}$$, which remains $$\Theta(n^3)$$. However, it does increase the constant factor in the running time and eliminates an elegant optimization that exploits the overlapping substructure of the $$w$$ computations themselves.
