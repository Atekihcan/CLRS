> ***Breaking a string***
>
> A certain string-processing language allows a programmer to break a string into two pieces. Because this operation copies the string, it costs $$n$$ time units to break a string of $$n$$ characters into two pieces. Suppose a programmer wants to break a string into many pieces. The order in which the breaks occur can affect the total amount of time used. For example, suppose that the programmer wants to break a 20-character string after characters 2, 8, and 10 (numbering the characters in ascending order from the left-hand end, starting from 1). If she programs the breaks to occur in left-to-right order, then the first break costs 20 time units, the second break costs 18 time units (breaking the string from characters 3 to 20 at character 8), and the third break costs 12 time units, totaling 50 time units. If she programs the breaks to occur in right-to-left order, however, then the first break costs 20 time units, the second break costs 10 time units, and the third break costs 8 time units, totaling 38 time units. In yet another order, she could break first at 8 (costing 20), then break the left piece at 2 (costing 8), and finally the right piece at 10 (costing 12), for a total cost of 40.
>
> Design an algorithm that, given the numbers of characters after which to break, determines a least-cost way to sequence those breaks. More formally, given a string $$S$$ with $$n$$ characters and an array $$L[1..m]$$ containing the break points, compute the lowest cost for a sequence of breaks, along with a sequence of breaks that achieves this cost.

This problem is structurally very similar to the matrix chain multiplication problem. Just as choosing different parenthesizations of matrices leads to different costs, choosing different orders for string breaks leads to different costs.

The key insight is that when we break a string, the cost depends on the current length of the piece we're breaking, not on its position in the original string. Once we make our first break, the two resulting pieces are independent subproblems.

### Setting Up the Subproblems

Let's add dummy break points at positions 0 and $$n$$ (the boundaries of the string). So our break points become $$B[0..m+1]$$ where $$B[0] = 0$$, $$B[1..m]$$ are the given break points (sorted), and $$B[m+1] = n$$.

Define $$c[i][j]$$ as the minimum cost to break the substring from position $$B[i]$$ to position $$B[j]$$ at all break points between them. The cost of breaking a string of length $$\ell$$ is $$\ell$$.

**Base case:** $$c[i][i+1] = 0$$ (no breaks to make between adjacent break points)

**Recurrence:** For $$j > i + 1$$, we must make at least one break. We try each possible first break at position $$B[k]$$ where $$i < k < j$$:

$$c[i][j] = (B[j] - B[i]) + \min_{i < k < j} \{ c[i][k] + c[k][j] \}$$

The term $$B[j] - B[i]$$ is the cost of making the current break (the length of the current substring).

### The Algorithm

{% capture code %}
Optimal-String-Break(L, m, n)
    // Set up break points array with boundaries
    let B[0..m+1] be a new array
    B[0] = 0
    for i = 1 to m
        B[i] = L[i]
    B[m+1] = n
    sort B[1..m]    // ensure break points are sorted

    // DP tables
    let c[0..m+1, 0..m+1] be a new table
    let root[0..m+1, 0..m+1] be a new table

    // Base cases: no breaks between adjacent points
    for i = 0 to m
        c[i][i+1] = 0

    // Fill table for increasing gap sizes
    for gap = 2 to m + 1
        for i = 0 to m + 1 - gap
            j = i + gap
            c[i][j] = infinity
            // Try each break point k between i and j
            for k = i + 1 to j - 1
                cost = c[i][k] + c[k][j]
                if cost < c[i][j]
                    c[i][j] = cost
                    root[i][j] = k
            // Add cost of this break
            c[i][j] = c[i][j] + (B[j] - B[i])

    return c[0][m+1], root

### Running Time Analysis

The break points array has $$m + 2$$ elements (including the boundaries). The DP table has $$(m + 2)^2 = O(m^2)$$ entries. Computing each entry requires trying $$O(m)$$ possible break points.

**Time complexity:** $$O(m^3)$$

**Space complexity:** $$O(m^2)$$ for the tables

### Example Walkthrough

For the example in the problem (string of length 20, breaks at 2, 8, 10):

Break points: $$B = [0, 2, 8, 10, 20]$$

The optimal solution is to break at 8 first (cost 20), then 2 (cost 8), then 10 (cost 12), for total cost 40.

Our algorithm finds this by computing:
- $$c[0][2] = 8$$ (break $$[0..8]$$ at 2)
- $$c[1][3] = 8$$ (break $$[2..10]$$ at 8)
- $$c[2][4] = 12$$ (break $$[8..20]$$ at 10)
- $$c[0][4] = 20 + \min(c[0][1]+c[1][4], c[0][2]+c[2][4], c[0][3]+c[3][4])$$

After evaluating all subproblems, $$c[0][4] = 40$$ with the break at 8 chosen first.
