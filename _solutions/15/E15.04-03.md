> Give a memoized version of $$\textsc{LCS-Length}$$ that runs in $$O(mn)$$ time.

Memoization is a top-down approach to dynamic programming where we start with the original problem and recursively break it down, but we cache (or "memo") the results of subproblems we've already solved. This avoids the exponential blowup of naive recursion while maintaining the natural recursive structure.

The strategy is straightforward: we create a table initialized with a special value (like $$-1$$) indicating "not yet computed." Before solving any subproblem, we check if it's already in the table. If so, we return the cached answer. If not, we compute it recursively, store the result, and return it.

{% capture code %}
Memoized-LCS-Length(X, Y)
    m = X.length
    n = Y.length
    let c[0..m, 0..n] be a new table
    for i = 0 to m
        for j = 0 to n
            c[i, j] = -1
    return Lookup-LCS(c, X, Y, m, n)
{% endcapture %}

{% capture code %}
Lookup-LCS(c, X, Y, i, j)
    if c[i, j] >= 0
        return c[i, j]
    if i == 0 or j == 0
        c[i, j] = 0
    elseif x_i == y_j
        c[i, j] = Lookup-LCS(c, X, Y, i - 1, j - 1) + 1
    else
        c[i, j] = max(Lookup-LCS(c, X, Y, i - 1, j),
                      Lookup-LCS(c, X, Y, i, j - 1))
    return c[i, j]
{% endcapture %}

The running time is $$O(mn)$$ because:

1. The table has $$(m + 1)(n + 1) = O(mn)$$ entries
2. Each entry is computed at most once (after that, we just return the cached value)
3. Computing each entry takes $$O(1)$$ time (excluding recursive calls)

Since every recursive call either returns immediately (if the value is cached) or computes a new table entry, and there are only $$O(mn)$$ table entries, the total work is $$O(mn)$$.

{% capture note %}
The memoized version and the bottom-up version have the same asymptotic running time of $$O(mn)$$. However, the bottom-up version typically has better constant factors because it avoids the overhead of recursive function calls. The memoized version has the advantage of only computing the subproblems that are actually needed, which can be beneficial if the recursion doesn't visit all subproblems. For the LCS problem, though, we typically need all subproblems, so the bottom-up approach is often preferred in practice.
{% endcapture %}
