> What is an optimal Huffman code for the following set of frequencies, based on the first 8 Fibonacci numbers?
>
> a:1 b:1 c:2 d:3 e:5 f:8 g:13 h:21
>
> Can you generalize your answer to find the optimal code when the frequencies are the first $$n$$ Fibonacci numbers?

Let's trace through the $$\textsc{Huffman}$$ algorithm on these frequencies. At each step, we extract the two nodes with minimum frequency and merge them.

**Step 1:** Merge a(1) and b(1) to get node with frequency 2.
Queue: {c:2, 2, d:3, e:5, f:8, g:13, h:21}

**Step 2:** Merge c(2) and (2) to get node with frequency 4.
Queue: {d:3, 4, e:5, f:8, g:13, h:21}

**Step 3:** Merge d(3) and (4) to get node with frequency 7.
Queue: {e:5, 7, f:8, g:13, h:21}

**Step 4:** Merge e(5) and (7) to get node with frequency 12.
Queue: {f:8, 12, g:13, h:21}

**Step 5:** Merge f(8) and (12) to get node with frequency 20.
Queue: {g:13, 20, h:21}

**Step 6:** Merge g(13) and (20) to get node with frequency 33.
Queue: {h:21, 33}

**Step 7:** Merge h(21) and (33) to get the root with frequency 54.

The resulting tree has a very specific structure: it's a completely unbalanced tree where each internal node has one leaf child and one internal child (except the deepest node which has two leaf children). The optimal Huffman code is:

| Character | Frequency | Codeword | Length |
|-----------|-----------|----------|--------|
| h | 21 | 0 | 1 |
| g | 13 | 10 | 2 |
| f | 8 | 110 | 3 |
| e | 5 | 1110 | 4 |
| d | 3 | 11110 | 5 |
| c | 2 | 111110 | 6 |
| b | 1 | 1111110 | 7 |
| a | 1 | 1111111 | 7 |

The total cost is: $$21(1) + 13(2) + 8(3) + 5(4) + 3(5) + 2(6) + 1(7) + 1(7) = 21 + 26 + 24 + 20 + 15 + 12 + 7 + 7 = 132$$ bits (per unit frequency).

### Generalization to $$n$$ Fibonacci Numbers

Let $$F_1, F_2, \ldots, F_n$$ denote the first $$n$$ Fibonacci numbers (with $$F_1 = F_2 = 1$$). The key observation is that for Fibonacci numbers, the sum of any two consecutive numbers equals the next: $$F_i + F_{i+1} = F_{i+2}$$.

When we run the Huffman algorithm:
- First, we merge $$F_1$$ and $$F_2$$ to get $$F_3$$
- Then we merge $$F_3$$ (the result) with the original $$F_3$$ to get $$F_4 + F_3 = F_5$$...

Actually, let's be more careful. After merging $$F_1$$ and $$F_2$$, we have frequency 2, and we need to compare with $$F_3 = 2$$. Since they're equal, we merge them to get 4. But $$F_4 = 3$$, so we merge 3 and 4 to get 7, and so on.

The pattern that emerges is that the optimal tree for Fibonacci frequencies is always a maximally unbalanced tree resembling a "spine" with leaves hanging off one side. The characters with larger frequencies get shorter codewords, with the two smallest-frequency characters sharing the longest codeword length.

For the first $$n$$ Fibonacci numbers, the optimal code assigns:
- The character with frequency $$F_n$$ gets codeword of length 1
- The character with frequency $$F_{n-1}$$ gets codeword of length 2
- ...
- The character with frequency $$F_2$$ gets codeword of length $$n-1$$
- The character with frequency $$F_1$$ gets codeword of length $$n-1$$

{% capture note %}
Fibonacci frequencies produce the worst-case scenario for Huffman coding in terms of tree balance. The resulting tree has maximum possible depth for $$n$$ leaves. This is related to the fact that Fibonacci numbers grow exponentially but as slowly as possible among sequences satisfying $$F_n = F_{n-1} + F_{n-2}$$.
