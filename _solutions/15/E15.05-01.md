> Write pseudocode for the procedure $$\textsc{Construct-Optimal-BST}(root)$$ which, given the table $$root$$, outputs the structure of an optimal binary search tree. For the example in Figure 15.10, your procedure should print out the structure
>
> k2 is the root
> k1 is the left child of k2
> d0 is the left child of k1
> d1 is the right child of k1
> k5 is the right child of k2
> k4 is the left child of k5
> k3 is the left child of k4
> d2 is the left child of k3
> d3 is the right child of k3
> d4 is the right child of k4
> d5 is the right child of k5
>
> corresponding to the optimal binary search tree shown in Figure 15.9(b).

Think of the $$root$$ table as a treasure map. Each entry $$root[i, j]$$ tells us which key $$k_r$$ should be the root when we're building the subtree containing keys $$k_i, k_{i+1}, \ldots, k_j$$. Starting from $$root[1, n]$$ gives us the overall root, and then we can recursively build left and right subtrees by looking up the appropriate entries.

The key insight is that once we know the root $$k_r$$ for a subtree spanning keys $$k_i$$ through $$k_j$$:
- The left subtree contains keys $$k_i, \ldots, k_{r-1}$$ (or just dummy key $$d_{i-1}$$ if $$r = i$$)
- The right subtree contains keys $$k_{r+1}, \ldots, k_j$$ (or just dummy key $$d_j$$ if $$r = j$$)

We need a recursive helper procedure that tracks the current range $$[i, j]$$ and its parent information. When $$j = i - 1$$, we've reached a dummy key $$d_{i-1}$$ (a leaf node).

{% capture code %}
Construct-Optimal-BST(root)
    Construct-Optimal-BST-Helper(root, 1, n, 0, "")

Construct-Optimal-BST-Helper(root, i, j, parent, direction)
    if j == i - 1
        // Reached a dummy key (leaf node)
        if parent == 0
            print "d" + (i-1) + " is the root"
        else
            print "d" + (i-1) + " is the " + direction + " child of k" + parent
    else
        // Get the root of this subtree from the table
        r = root[i, j]
        if parent == 0
            print "k" + r + " is the root"
        else
            print "k" + r + " is the " + direction + " child of k" + parent
        // Recursively process left and right subtrees
        Construct-Optimal-BST-Helper(root, i, r - 1, r, "left")
        Construct-Optimal-BST-Helper(root, r + 1, j, r, "right")
{% endcapture %}

The procedure works through a preorder traversal of the optimal BST. We start at the overall root (obtained from $$root[1, n]$$), print it, then recursively process the left subtree and right subtree. The base case handles dummy keys, which are leaves in the tree.

Let's trace through the example from Figure 15.10 where $$n = 5$$:

1. Call $$\textsc{Construct-Optimal-BST-Helper}(root, 1, 5, 0, "")$$
   - $$root[1, 5] = 2$$, so print "k2 is the root"
   - Recurse left with $$(1, 1, 2, "left")$$ and right with $$(3, 5, 2, "right")$$

2. Processing left child $$(1, 1, 2, "left")$$:
   - $$root[1, 1] = 1$$, print "k1 is the left child of k2"
   - Recurse left with $$(1, 0, 1, "left")$$ and right with $$(2, 1, 1, "right")$$

3. Processing $$(1, 0, 1, "left")$$:
   - $$j = 0 < i = 1$$, so this is dummy key $$d_0$$
   - Print "d0 is the left child of k1"

4. Processing $$(2, 1, 1, "right")$$:
   - $$j = 1 < i = 2$$, so this is dummy key $$d_1$$
   - Print "d1 is the right child of k1"

5. Processing right child $$(3, 5, 2, "right")$$:
   - $$root[3, 5] = 5$$, print "k5 is the right child of k2"
   - Continue recursively...

This produces exactly the output shown in the problem statement.

{% capture note %}
The running time is $$\Theta(n)$$ since we visit each of the $$n$$ keys exactly once and each of the $$n + 1$$ dummy keys exactly once. At each node, we do constant work (a table lookup and a print statement), so the total time is proportional to the number of nodes, which is $$2n + 1$$.
{% endcapture %}
