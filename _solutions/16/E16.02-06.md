> $$\star$$ Show how to solve the fractional knapsack problem in $$O(n)$$ time.

The standard greedy algorithm for fractional knapsack sorts items by value-per-weight ratio, taking $$O(n \log n)$$ time. To achieve $$O(n)$$ time, we avoid full sorting by using the linear-time median-finding algorithm from Chapter 9.

The key observation is that we don't need items in sorted order. We just need to find the "critical item" that straddles the knapsack capacity (the item we might take fractionally). All items with higher value-per-weight ratios go in completely; all items with lower ratios stay out.

{% capture code %}
Fast-Fractional-Knapsack(items, W)
    // Each item has weight w and value v
    n = items.length
    if n == 0
        return 0

    // Compute ratios r[i] = v[i] / w[i]
    for each item i
        r[i] = v[i] / w[i]

    // Use SELECT to find median ratio in O(n) time
    median = Select(r, ⌊n/2⌋)

    // Partition items by ratio
    L = {i : r[i] > median}      // high-ratio items
    M = {i : r[i] = median}      // median-ratio items
    H = {i : r[i] < median}      // low-ratio items

    W_L = sum of w[i] for i in L

    if W_L > W
        // Answer lies entirely in high-ratio items
        return Fast-Fractional-Knapsack(L, W)
    else
        // Take all of L; recurse on remainder
        V_L = sum of v[i] for i in L
        W_remaining = W - W_L
        W_M = sum of w[i] for i in M

        if W_L + W_M >= W
            // Critical item is in M (all have same ratio)
            return V_L + W_remaining × median
        else
            // Take all of L and M; recurse on low-ratio items
            V_M = sum of v[i] for i in M
            return V_L + V_M + Fast-Fractional-Knapsack(H, W_remaining - W_M)
{% endcapture %}

**Analysis:** Let $$T(n)$$ be the running time. Each recursive call:
1. Computes ratios in $$O(n)$$ time.
2. Finds median using $$\textsc{Select}$$ in $$O(n)$$ time.
3. Partitions items in $$O(n)$$ time.
4. Recurses on at most half the items.

This gives the recurrence:

$$T(n) = T(n/2) + O(n)$$

By the master theorem (case 3 with $$a = 1$$, $$b = 2$$, $$f(n) = n$$), we get $$T(n) = O(n)$$.

**Correctness:** The algorithm correctly identifies which items to take fully and which to exclude. When $$W_L > W$$, the knapsack fills entirely with high-ratio items. When $$W_L + W_M \geq W$$, we take all high-ratio items plus a fraction of median-ratio items. Otherwise, we take all high and median items, then recurse on lower-ratio items.

{% capture note %}
The $$O(n)$$ selection algorithm (from Chapter 9) is the key enabler. Without it, finding the median would require sorting, defeating the purpose. The technique of "prune and search" (eliminating half the candidates each round) appears throughout algorithm design, from binary search to linear-time selection to this knapsack solution.
