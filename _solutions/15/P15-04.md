> ***Printing neatly***
>
> Consider the problem of neatly printing a paragraph with a monospaced font (all characters having the same width) on a printer. The input text is a sequence of $$n$$ words of lengths $$l_1, l_2, \ldots, l_n$$, measured in characters. We want to print this paragraph neatly on a number of lines that hold a maximum of $$M$$ characters each. Our criterion of "neatness" is as follows. If a given line contains words $$i$$ through $$j$$, where $$i \leq j$$, and we leave exactly one space between words, the number of extra space characters at the end of the line is $$M - j + i - \sum_{k=i}^{j} l_k$$, which must be nonnegative so that the words fit on the line. We wish to minimize the sum, over all lines except the last, of the cubes of the numbers of extra space characters at the ends of lines. Give a dynamic-programming algorithm to print a paragraph of $$n$$ words neatly on a printer. Analyze the running time and space requirements of your algorithm.

This problem is about making text look nice, similar to what word processors do when they justify paragraphs. Using the cube of the extra space (rather than the space itself) penalizes unevenness more heavily. A line with 6 extra spaces contributes $$6^3 = 216$$, whereas two lines with 3 spaces each contribute only $$2 \times 27 = 54$$. This encourages distributing slack more evenly.

### Computing Extra Space

First, let's compute how much extra space remains if we put words $$i$$ through $$j$$ on a single line. The words occupy $$\sum_{k=i}^{j} l_k$$ characters, and we need $$j - i$$ spaces between them. So:

$$\text{extras}[i][j] = M - (j - i) - \sum_{k=i}^{j} l_k$$

If $$\text{extras}[i][j] < 0$$, the words don't fit on one line.

We define the line cost as:

$$\text{lc}[i][j] = \begin{cases}
\infty & \text{if words } i..j \text{ don't fit (extras} < 0\text{)} \\
0 & \text{if } j = n \text{ and words fit (last line)} \\
(\text{extras}[i][j])^3 & \text{otherwise}
\end{cases}$$

The last line is free because we don't care about trailing whitespace at the end of a paragraph.

### The Dynamic Programming Solution

Let $$c[j]$$ be the minimum cost of printing words $$1$$ through $$j$$. The recurrence is:

$$c[j] = \min_{1 \leq i \leq j} \{ c[i-1] + \text{lc}[i][j] \}$$

with base case $$c[0] = 0$$ (no words means no cost).

The intuition is straightforward: to find the best way to print words $$1$$ through $$j$$, we try every possible last line (starting at word $$i$$ and ending at word $$j$$), and pick the one that minimizes the total cost.

{% capture code %}
Print-Neatly(l, n, M)
    // Compute extras[i][j] for all valid i,j
    let extras[1..n, 1..n] be a new table
    for i = 1 to n
        extras[i][i] = M - l[i]
        for j = i + 1 to n
            extras[i][j] = extras[i][j-1] - l[j] - 1

    // Compute line costs lc[i][j]
    let lc[1..n, 1..n] be a new table
    for i = 1 to n
        for j = i to n
            if extras[i][j] < 0
                lc[i][j] = infinity
            elseif j == n
                lc[i][j] = 0
            else
                lc[i][j] = (extras[i][j])^3

    // Compute optimal costs c[j] and breaks p[j]
    let c[0..n] and p[1..n] be new arrays
    c[0] = 0
    for j = 1 to n
        c[j] = infinity
        for i = 1 to j
            if c[i-1] + lc[i][j] < c[j]
                c[j] = c[i-1] + lc[i][j]
                p[j] = i

    return c[n] and p
{% endcapture %}

### Reconstructing the Solution

The array $$p[j]$$ stores the starting word of the last line when printing words $$1$$ through $$j$$. To print the actual line breaks:

Print-Lines(p, j)

### Running Time and Space Analysis

**Time complexity:** Computing the extras table takes $$O(n^2)$$. Computing the line costs takes $$O(n^2)$$. The main DP loop computes $$c[j]$$ for $$j = 1$$ to $$n$$, and each computation examines up to $$j$$ previous values, giving $$O(n^2)$$ total. Thus, the overall time is $$O(n^2)$$.

**Space complexity:** We store the extras table ($$O(n^2)$$), line costs table ($$O(n^2)$$), the $$c$$ array ($$O(n)$$), and the $$p$$ array ($$O(n)$$). Total space is $$O(n^2)$$.

{% capture note %}
We can reduce space to $$O(n)$$ by observing that we don't need to store the entire extras and lc tables. When computing $$c[j]$$, we can compute $$\text{extras}[i][j]$$ on the fly for each $$i$$, starting from $$i = j$$ and working backwards. This requires $$O(1)$$ extra work per $$i$$ value, maintaining the same $$O(n^2)$$ time complexity.
{% endcapture %}
