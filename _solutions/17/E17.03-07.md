> Design a data structure to support the following two operations for a dynamic multiset $$S$$ of integers, which allows duplicate values:
>
> $$\textsc{Insert}(S, x)$$ inserts $$x$$ into $$S$$.
>
> $$\textsc{Delete-Larger-Half}(S)$$ deletes the largest $$\lceil |S|/2 \rceil$$ elements from $$S$$.
>
> Explain how to implement this data structure so that any sequence of $$m$$ $$\textsc{Insert}$$ and $$\textsc{Delete-Larger-Half}$$ operations runs in $$O(m)$$ time. Your implementation should also include a way to output the elements of $$S$$ in $$O(|S|)$$ time.

At first glance, $$\textsc{Delete-Larger-Half}$$ seems expensive since we need to find and remove roughly half of the elements. The key insight is that every element deleted was previously inserted, so we can "charge" each deletion to the insertion that created it.

We implement $$S$$ as an unsorted array (or doubly linked list) with a count of the number of elements.

**$$\textsc{Insert}(S, x)$$:** Append $$x$$ to the end of the array. Cost: $$O(1)$$.

**$$\textsc{Delete-Larger-Half}(S)$$:** Let $$n = |S|$$ and $$k = \lceil n/2 \rceil$$. We need to find and delete the $$k$$ largest elements. Use a linear-time selection algorithm (like $$\textsc{Select}$$ from Chapter 9) to find the $$k$$th largest element, then partition the array around this element and remove all elements in the larger half. Cost: $$O(n)$$.

**$$\textsc{Output}(S)$$:** Simply traverse the array. Cost: $$O(|S|)$$.

To show that $$m$$ operations take $$O(m)$$ time, we use the potential method with:

$$\Phi(S) = 2|S|$$

Initially, $$S$$ is empty, so $$\Phi(S_0) = 0$$. Since $$|S| \ge 0$$, we have $$\Phi(S_i) \ge 0$$.

**Amortized cost of $$\textsc{Insert}$$:**

Inserting one element costs 1 and increases $$|S|$$ by 1.

$$\hat{c}_{\textsc{Insert}} = 1 + 2(|S| + 1) - 2|S| = 3$$

**Amortized cost of $$\textsc{Delete-Larger-Half}$$:**

Let $$n = |S|$$ before the operation and $$k = \lceil n/2 \rceil$$ elements deleted. The actual cost is $$O(n)$$, say $$cn$$ for some constant $$c$$. After the operation, $$|S| = n - k = n - \lceil n/2 \rceil = \lfloor n/2 \rfloor$$.

$$\begin{align*}
\hat{c}_{\textsc{Delete}} &= cn + 2\lfloor n/2 \rfloor - 2n \\
                          &\le cn + n - 2n \\
                          &= cn - n \\
                          &= (c - 1)n
\end{align*}$$

Hmm, this is still $$O(n)$$, not $$O(1)$$. Let us increase the potential. Define:

$$\Phi(S) = c' \cdot |S|$$

where $$c'$$ is chosen large enough. With $$k \ge n/2$$:

$$\begin{align*}
\hat{c}_{\textsc{Delete}} &= cn + c'(n - k) - c'n \\
                          &= cn - c' k \\
                          &\le cn - c' \cdot \frac{n}{2}
\end{align*}$$

Choosing $$c' = 2c$$, we get:

$$\hat{c}_{\textsc{Delete}} \le cn - cn = 0$$

So with $$\Phi(S) = 2c|S|$$ (where $$c$$ is the constant in the $$O(n)$$ cost of selection):

- $$\hat{c}_{\textsc{Insert}} = 1 + 2c = O(1)$$
- $$\hat{c}_{\textsc{Delete}} \le 0$$

Since both amortized costs are $$O(1)$$, the total cost of $$m$$ operations is $$O(m)$$.

The essential intuition is that each element can only be deleted once, and deleting it "releases" the potential that was stored when it was inserted. Since we delete at least half the elements each time, the decrease in potential is enough to pay for the linear-time selection.
