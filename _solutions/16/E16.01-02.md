> Suppose that instead of always selecting the first activity to finish, we instead select the last activity to start that is compatible with all previously selected activities. Describe how this approach is a greedy algorithm, and prove that it yields an optimal solution.

The original greedy strategy selects the activity with the earliest finish time, which leaves as much room as possible for future activities. The proposed alternative works from the other direction: by selecting the activity with the latest start time, we leave as much room as possible for preceding activities.

Think of it like scheduling appointments in your calendar. The original approach fills your day starting from the earliest slot. This alternative fills your day starting from the latest slot. Both approaches maximize how many appointments fit.

### The Algorithm

We assume activities are sorted in monotonically decreasing order of start time (or equivalently, we process them from latest to earliest start time). The algorithm selects the last activity to start, then recursively selects from activities that finish before the selected activity starts.

{% capture code %}
Greedy-Activity-Selector-Latest-Start(s, f)
    n = s.length
    A = {a_n}                        // activity with latest start time
    k = n
    for m = n - 1 downto 1
        if f[m] <= s[k]              // a_m finishes before a_k starts
            A = A ∪ {a_m}
            k = m
    return A
{% endcapture %}

### Proof of Optimality

We prove that this greedy choice always leads to an optimal solution using an exchange argument similar to Theorem 16.1 in the text.

**Theorem:** Let $$S_k$$ be a nonempty subproblem, and let $$a_m$$ be the activity in $$S_k$$ with the latest start time. Then $$a_m$$ is included in some maximum-size subset of mutually compatible activities of $$S_k$$.

**Proof:** Let $$A_k$$ be a maximum-size subset of mutually compatible activities in $$S_k$$, and let $$a_j$$ be the activity in $$A_k$$ with the latest start time. If $$a_j = a_m$$, we are done.

If $$a_j \neq a_m$$, construct $$A'_k = A_k - \{a_j\} \cup \{a_m\}$$. The activities in $$A'_k$$ are mutually compatible because:
- All activities in $$A_k$$ except $$a_j$$ are compatible with each other
- Since $$a_m$$ has the latest start time in $$S_k$$ and $$a_j$$ was the latest-starting activity in $$A_k$$, we have $$s_m \ge s_j$$
- Any activity that finishes before $$a_j$$ starts also finishes before $$a_m$$ starts (since $$s_m \ge s_j$$)

Since $$|A'_k| = |A_k|$$, the set $$A'_k$$ is also a maximum-size subset, and it contains $$a_m$$. Therefore, there exists an optimal solution containing the greedy choice.

### Why This Works

The symmetry between the two approaches reflects a deeper truth about the activity-selection problem. The problem has a natural time-reversal symmetry: if you reverse time (swap start and finish times for all activities), selecting the earliest finish time in the original problem becomes selecting the latest start time in the reversed problem. Both strategies exploit the same greedy-choice property, just from opposite ends of the timeline.

{% capture note %}
This is analogous to how Dijkstra's algorithm can find shortest paths both forward from a source and backward from a destination. The structure of the problem allows optimal solutions to be built in either direction, and the greedy choice is simply the symmetric counterpart when we reverse our perspective.
