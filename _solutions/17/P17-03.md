> ***Amortized weight-balanced trees***
>
> Consider an ordinary binary search tree augmented by adding to each node $$x$$ the attribute $$x.\text{size}$$ giving the number of keys stored in the subtree rooted at $$x$$. Let $$\alpha$$ be a constant in the range $$1/2 \le \alpha < 1$$. We say that a given node $$x$$ is $$\alpha$$-balanced if $$x.\text{left}.\text{size} \le \alpha \cdot x.\text{size}$$ and $$x.\text{right}.\text{size} \le \alpha \cdot x.\text{size}$$. The tree as a whole is $$\alpha$$-balanced if every node in the tree is $$\alpha$$-balanced. The following amortized approach to maintaining weight-balanced trees was suggested by G. Varghese.
>
> 1. A $$1/2$$-balanced tree is, in a sense, as balanced as it can be. Given a node $$x$$ in an arbitrary binary search tree, show how to rebuild the subtree rooted at $$x$$ so that it becomes $$1/2$$-balanced. Your algorithm should run in time $$\Theta(x.\text{size})$$, and it can use $$O(x.\text{size})$$ auxiliary storage.
>
> 2. Show that performing a search in an $$n$$-node $$\alpha$$-balanced binary search tree takes $$O(\lg n)$$ worst-case time.
>
> For the remainder of this problem, assume that the constant $$\alpha$$ is strictly greater than $$1/2$$. Suppose that we implement $$\textsc{Insert}$$ and $$\textsc{Delete}$$ as usual for an $$n$$-node binary search tree, except that after every such operation, if any node in the tree is no longer $$\alpha$$-balanced, then we "rebuild" the subtree rooted at the highest such node in the tree so that it becomes $$1/2$$-balanced.
>
> We shall analyze this rebuilding scheme using the potential method. For a node $$x$$ in a binary search tree $$T$$, we define $$\Delta(x) = |x.\text{left}.\text{size} - x.\text{right}.\text{size}|$$, and we define the potential of $$T$$ as $$\Phi(T) = c \sum_{x \in T : \Delta(x) \ge 2} \Delta(x)$$, where $$c$$ is a sufficiently large constant that depends on $$\alpha$$.
>
> 3. Argue that any binary search tree has nonnegative potential and that a $$1/2$$-balanced tree has potential 0.
>
> 4. Suppose that $$m$$ units of potential can pay for rebuilding an $$m$$-node subtree. How large must $$c$$ be in terms of $$\alpha$$ in order for it to take $$O(1)$$ amortized time to rebuild a subtree that is not $$\alpha$$-balanced?
>
> 5. Show that inserting a node into or deleting a node from an $$n$$-node $$\alpha$$-balanced tree costs $$O(\lg n)$$ amortized time.

Weight-balanced trees provide an alternative to height-balanced trees (like AVL or red-black trees) for maintaining balance in binary search trees. The $$\alpha$$-balance condition requires that neither child's subtree contains more than fraction $$\alpha$$ of the nodes. When $$\alpha = 1/2$$, the tree is perfectly balanced; when $$\alpha$$ approaches 1, the tree can become arbitrarily unbalanced.

### A

To rebuild a subtree rooted at $$x$$ into a $$1/2$$-balanced tree, we use a two-phase approach:

**Phase 1:** Flatten the subtree into a sorted array using an in-order traversal. This takes $$\Theta(x.\text{size})$$ time and uses $$O(x.\text{size})$$ auxiliary storage for the array.

**Phase 2:** Recursively build a balanced BST from the sorted array by always choosing the middle element as the root. This naturally creates a $$1/2$$-balanced tree.

{% capture code %}
Rebuild(x)
    let A[1..x.size] be a new array
    Flatten(x, A, 1)
    return Build-Balanced(A, 1, x.size)

Flatten(x, A, i)
    if x == NIL
        return i
    i = Flatten(x.left, A, i)
    A[i] = x
    i = i + 1
    i = Flatten(x.right, A, i)
    return i

Build-Balanced(A, lo, hi)
    if lo > hi
        return NIL
    mid = floor((lo + hi) / 2)
    x = A[mid]
    x.left = Build-Balanced(A, lo, mid - 1)
    x.right = Build-Balanced(A, mid + 1, hi)
    x.size = hi - lo + 1
    return x
{% endcapture %}

The $$\textsc{Flatten}$$ procedure visits each node exactly once, taking $$\Theta(n)$$ time for $$n$$ nodes. The $$\textsc{Build-Balanced}$$ procedure also visits each position in the array exactly once, taking $$\Theta(n)$$ time. The resulting tree is $$1/2$$-balanced because at each node, we choose the median element, ensuring that the left and right subtrees differ in size by at most 1.

### B

In an $$\alpha$$-balanced tree, each node $$x$$ satisfies $$x.\text{left}.\text{size} \le \alpha \cdot x.\text{size}$$ and $$x.\text{right}.\text{size} \le \alpha \cdot x.\text{size}$$. This means when we descend from a node to either child, the subtree size is at most $$\alpha$$ times the current subtree size.

Starting from the root with $$n$$ nodes, after $$h$$ steps along any path, the subtree size is at most $$\alpha^h \cdot n$$. A subtree must have at least 1 node, so we need:

$$\alpha^h \cdot n \ge 1$$

Taking logarithms:

$$\begin{align*}
h \cdot \lg \alpha &\ge -\lg n \\
h &\le \frac{\lg n}{-\lg \alpha} \\
h &= O\left(\frac{\lg n}{\lg(1/\alpha)}\right) \\
h &= O(\lg n)
\end{align*}$$

Since $$\alpha$$ is a constant with $$1/2 \le \alpha < 1$$, the term $$\lg(1/\alpha)$$ is a positive constant. Therefore, the height of an $$\alpha$$-balanced tree is $$O(\lg n)$$, and search takes $$O(\lg n)$$ time in the worst case.

{% capture note %}
For $$\alpha = 1/2$$, we get $$h \le \lg n$$, which is the height of a perfectly balanced tree. For $$\alpha = 3/4$$, we get $$h \le \lg n / \lg(4/3) \approx 2.4 \lg n$$. The tree is still logarithmically deep, just with a larger constant factor.
{% endcapture %}

### C

The potential function is $$\Phi(T) = c \sum_{x \in T : \Delta(x) \ge 2} \Delta(x)$$.

**Nonnegative potential:** The sum is over non-negative terms (since $$\Delta(x) = |x.\text{left}.\text{size} - x.\text{right}.\text{size}| \ge 0$$), and $$c > 0$$, so $$\Phi(T) \ge 0$$ for any tree $$T$$.

**Zero potential for $$1/2$$-balanced trees:** In a $$1/2$$-balanced tree created by the rebuild procedure, each node $$x$$ has $$|x.\text{left}.\text{size} - x.\text{right}.\text{size}| \le 1$$. Specifically, when we build from a sorted array by choosing the middle element, the left and right subtrees differ in size by at most 1. Since $$\Delta(x) \le 1 < 2$$ for all nodes, no node contributes to the sum, so $$\Phi(T) = 0$$.

### D

When a node $$x$$ becomes not $$\alpha$$-balanced, we have (without loss of generality) $$x.\text{left}.\text{size} > \alpha \cdot x.\text{size}$$. Since the subtree sizes sum to $$x.\text{size} - 1$$ (excluding $$x$$ itself), we have:

$$x.\text{right}.\text{size} = x.\text{size} - 1 - x.\text{left}.\text{size} < x.\text{size} - 1 - \alpha \cdot x.\text{size} = (1 - \alpha) \cdot x.\text{size} - 1$$

The imbalance is:

$$\begin{align*}
\Delta(x) &= x.\text{left}.\text{size} - x.\text{right}.\text{size} \\
&> \alpha \cdot x.\text{size} - ((1 - \alpha) \cdot x.\text{size} - 1) \\
&= (2\alpha - 1) \cdot x.\text{size} + 1
\end{align*}$$

Since $$\alpha > 1/2$$, we have $$2\alpha - 1 > 0$$. Let $$m = x.\text{size}$$. The potential at node $$x$$ alone is at least:

$$c \cdot \Delta(x) > c \cdot (2\alpha - 1) \cdot m$$

For the potential released to pay for the $$O(m)$$ rebuilding cost, we need:

$$c \cdot (2\alpha - 1) \cdot m \ge m$$

This gives us:

$$c \ge \frac{1}{2\alpha - 1}$$

For example, if $$\alpha = 2/3$$, then $$c \ge 1/(2 \cdot 2/3 - 1) = 1/(1/3) = 3$$.

### E

Consider an insertion or deletion operation. We first perform the standard BST operation in $$O(\lg n)$$ time (since the tree is $$\alpha$$-balanced, it has $$O(\lg n)$$ height by part B). Then we check for balance violations and potentially rebuild.

**Insertion case:** Inserting a node increases the size of all ancestors by 1 and may increase their $$\Delta$$ values by at most 1. There are $$O(\lg n)$$ ancestors, so the potential increases by at most $$O(c \lg n) = O(\lg n)$$.

If no rebuilding is needed, the amortized cost is:

$$\hat{c} = O(\lg n) + O(\lg n) = O(\lg n)$$

If rebuilding is needed at node $$x$$, the actual cost is $$O(\lg n) + O(m)$$ where $$m = x.\text{size}$$. But by part D, the potential released is at least $$m$$, so:

$$\hat{c} = O(\lg n) + O(m) - m + O(\lg n) = O(\lg n)$$

**Deletion case:** Similar analysis applies. Deleting a node changes $$\Delta$$ values along the path by at most 1 each. If rebuilding occurs, the released potential pays for it.

**Key insight:** After a subtree is rebuilt to be perfectly balanced ($$\Phi = 0$$ for that subtree), it takes $$\Omega(m)$$ insertions or deletions affecting that subtree before it becomes $$\alpha$$-unbalanced again. Each such operation contributes $$O(1)$$ to the potential, so by the time rebuilding is needed, sufficient potential has accumulated.

Therefore, both insertion and deletion take $$O(\lg n)$$ amortized time.
