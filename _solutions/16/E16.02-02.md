> Give a dynamic-programming solution to the 0-1 knapsack problem that runs in $$O(nW)$$ time, where $$n$$ is the number of items and $$W$$ is the maximum weight of items that the thief can put in his knapsack.

The 0-1 knapsack problem asks us to select a subset of $$n$$ items, each with weight $$w_i$$ and value $$v_i$$, that maximizes total value while staying within weight capacity $$W$$. Unlike the fractional version, we must take items whole or not at all.

The key observation is that for each item, we have exactly two choices: include it or exclude it. This binary decision leads naturally to a dynamic programming formulation. Let $$K[i, w]$$ denote the maximum value achievable using items $$1, 2, \ldots, i$$ with knapsack capacity $$w$$.

For item $$i$$, if its weight $$w_i > w$$, we cannot include it, so $$K[i, w] = K[i-1, w]$$. Otherwise, we take the better of two options: exclude item $$i$$ (getting value $$K[i-1, w]$$) or include it (getting value $$v_i + K[i-1, w - w_i]$$). This gives us:

$$K[i, w] = \begin{cases}
0 & \text{if } i = 0 \text{ or } w = 0 \\
K[i-1, w] & \text{if } w_i > w \\
\max(K[i-1, w], \; v_i + K[i-1, w - w_i]) & \text{otherwise}
\end{cases}$$

{% capture code %}
Zero-One-Knapsack(v, w, n, W)
    let K[0..n, 0..W] be a new table
    for w = 0 to W
        K[0, w] = 0
    for i = 1 to n
        K[i, 0] = 0
        for w = 1 to W
            if w[i] > w
                K[i, w] = K[i - 1, w]
            else
                K[i, w] = max(K[i - 1, w], v[i] + K[i - 1, w - w[i]])
    return K[n, W]
{% endcapture %}
{% include clrs_code.html title="Zero-One-Knapsack(v, w, n, W)" %}

The algorithm fills a table of size $$(n+1) \times (W+1)$$, computing each entry in $$O(1)$$ time. The total running time is $$O(nW)$$.

To reconstruct the actual items selected, we can trace back through the table: if $$K[i, w] \neq K[i-1, w]$$, then item $$i$$ was included, and we recurse on $$K[i-1, w - w_i]$$.

{% capture note %}
The $$O(nW)$$ running time is called "pseudo-polynomial" because $$W$$ appears in the complexity directly, not as $$\log W$$. If $$W$$ is exponentially large in the input size, this algorithm becomes exponential. This is why the 0-1 knapsack problem remains NP-complete even with this dynamic programming solution.
{% endcapture %}
