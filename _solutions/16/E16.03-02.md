> Prove that a binary tree that is not full cannot correspond to an optimal prefix code.

A full binary tree is one where every internal node has exactly two children. We'll prove that if a binary tree representing a prefix code is not full, we can always construct a better code by modifying the tree.

Suppose we have a binary tree $$T$$ that represents a prefix code, and $$T$$ is not full. This means there exists some internal node $$v$$ in $$T$$ that has only one child. Let's call this child $$u$$ (which could be either a leaf or another internal node).

Consider what this structure means for the code. The node $$v$$ contributes to the path length of every character in the subtree rooted at $$u$$, but it only has one child, so the bit used at $$v$$ provides no actual disambiguation. Every codeword that passes through $$v$$ contains an "unnecessary" bit.

We can construct a strictly better tree $$T'$$ by removing node $$v$$ and replacing it directly with the subtree rooted at $$u$$. In other words, we make $$u$$ a child of $$v$$'s parent (or make $$u$$ the root if $$v$$ was the root). This operation reduces the depth of every leaf in the subtree rooted at $$u$$ by exactly 1.

Let $$S$$ be the set of characters (leaves) in the subtree rooted at $$u$$. The cost reduction is:

$$B(T) - B(T') = \sum_{c \in S} c.\text{freq} \cdot d_T(c) - \sum_{c \in S} c.\text{freq} \cdot d_{T'}(c) = \sum_{c \in S} c.\text{freq}$$

Since all frequencies are positive (we only consider characters that actually appear in the file), this sum is strictly positive. Therefore $$B(T') < B(T)$$, meaning $$T$$ was not optimal.

This argument can be applied repeatedly: if $$T'$$ still has a node with only one child, we can remove it too. Each removal strictly decreases the cost. Since the tree is finite, this process must terminate, and the final tree will be full.

{% capture note %}
This result connects directly to information theory. In a full binary tree, every bit in a codeword helps distinguish between characters. A non-full node represents a wasted bit, which corresponds to inefficiency in the code. The Huffman algorithm naturally produces full binary trees because it always creates internal nodes with exactly two children (the two nodes being merged).
