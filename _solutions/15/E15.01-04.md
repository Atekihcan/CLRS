> Modify $$\textsc{Memoized-Cut-Rod}$$ to return not only the value but the actual solution, too.

The original $$\textsc{Memoized-Cut-Rod}$$ only tells us the maximum revenue, not how to achieve it. To reconstruct the actual cutting strategy, we need to remember which choice led to the optimal value at each step. This is a common pattern in dynamic programming: alongside the optimal value, we store the decision that achieved it.

We'll add an auxiliary array $$s[0..n]$$ where $$s[j]$$ records the optimal first piece length when solving the subproblem for a rod of length $$j$$. Once we have this array, we can trace back through it to reconstruct the complete solution.

{% capture code %}
Memoized-Cut-Rod-Extended(p, n)
    let r[0..n] and s[0..n] be new arrays
    for i = 0 to n
        r[i] = -infinity
    Memoized-Cut-Rod-Aux-Extended(p, n, r, s)
    return r and s
{% endcapture %}
{% include clrs_code.html title="Memoized-Cut-Rod-Extended(p, n)" %}

{% capture code %}
Memoized-Cut-Rod-Aux-Extended(p, n, r, s)
    if r[n] >= 0
        return r[n]
    if n == 0
        q = 0
    else
        q = -infinity
        for i = 1 to n
            val = p[i] + Memoized-Cut-Rod-Aux-Extended(p, n - i, r, s)
            if val > q
                q = val
                s[n] = i
    r[n] = q
    return q
{% endcapture %}

To print the actual solution, we trace through the $$s$$ array:

{% capture code %}
Print-Cut-Rod-Solution(p, n)
    (r, s) = Memoized-Cut-Rod-Extended(p, n)
    while n > 0
        print s[n]
        n = n - s[n]
{% endcapture %}
{% include clrs_code.html title="Print-Cut-Rod-Solution(p, n)" %}

The printing procedure works by repeatedly looking up the optimal first piece size for the current remaining length, printing it, and then moving to the subproblem for the remainder.

{% capture note %}
The modification adds only $$O(n)$$ extra space for the $$s$$ array and constant extra work per subproblem. The asymptotic running time remains $$\Theta(n^2)$$, the same as the original memoized version. This pattern of storing "which choice was optimal" alongside "what was the optimal value" is fundamental to dynamic programming. The textbook's $$\textsc{Extended-Bottom-Up-Cut-Rod}$$ uses the same idea in the bottom-up approach.
{% endcapture %}
