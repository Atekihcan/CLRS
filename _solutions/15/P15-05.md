> ***Edit distance***
>
> In order to transform one source string of text $$x[1..m]$$ to a target string $$y[1..n]$$, we can perform various transformation operations. Our goal is, given $$x$$ and $$y$$, to produce a series of transformations that change $$x$$ to $$y$$. We use an array $$z$$ to hold the intermediate results. Initially, $$z$$ is empty, and at termination, we should have $$z[j] = y[j]$$ for $$j = 1, 2, \ldots, n$$. We maintain current indices $$i$$ into $$x$$ and $$j$$ into $$z$$, and the operations are allowed to alter $$z$$ and these indices. Initially, $$i = j = 1$$. We are required to examine every character in $$x$$ during the transformation, which means that at the end of the sequence of transformation operations, we must have $$i = m + 1$$.
>
> We may choose from among six transformation operations:
>
> - **Copy** a character from $$x$$ to $$z$$ by setting $$z[j] = x[i]$$ and then incrementing both $$i$$ and $$j$$.
> - **Replace** a character from $$x$$ by another character $$c$$, by setting $$z[j] = c$$, and then incrementing both $$i$$ and $$j$$.
> - **Delete** a character from $$x$$ by incrementing $$i$$ but leaving $$j$$ alone.
> - **Insert** the character $$c$$ into $$z$$ by setting $$z[j] = c$$ and then incrementing $$j$$, but leaving $$i$$ alone.
> - **Twiddle** (i.e., exchange) the next two characters by copying them from $$x$$ to $$z$$ but in the opposite order.
> - **Kill** the remainder of $$x$$ by setting $$i = m + 1$$. This operation, if performed, must be the final operation.
>
> 1. Given two sequences $$x[1..m]$$ and $$y[1..n]$$ and set of transformation-operation costs, the edit distance from $$x$$ to $$y$$ is the cost of the least expensive operation sequence that transforms $$x$$ to $$y$$. Describe a dynamic-programming algorithm that finds the edit distance from $$x[1..m]$$ to $$y[1..n]$$ and prints an optimal operation sequence. Analyze the running time and space requirements of your algorithm.
>
> 2. Explain how to cast the problem of finding an optimal alignment as an edit distance problem using a subset of the transformation operations copy, replace, delete, insert, twiddle, and kill.

The edit distance problem measures how different two strings are by counting the minimum cost of operations needed to transform one into the other. This has applications ranging from spell checkers to bioinformatics.

### A. Dynamic Programming Algorithm

The key insight is that the state of our transformation can be characterized by two indices: how far we've processed in the source string $$x$$ (index $$i$$), and how much of the target string $$z$$ we've built (index $$j$$, which should match prefix of $$y$$).

Let $$c[i][j]$$ be the minimum cost to transform $$x[1..i]$$ into $$y[1..j]$$. Our goal is to compute $$c[m][n]$$.

The recurrence considers what operation we might have performed last:

$$c[i][j] = \min \begin{cases}
c[i-1][j-1] + \text{cost}(\text{copy}) & \text{if } x[i] = y[j] \\
c[i-1][j-1] + \text{cost}(\text{replace}) & \text{(replace } x[i] \text{ with } y[j]\text{)} \\
c[i-1][j] + \text{cost}(\text{delete}) & \text{(delete } x[i]\text{)} \\
c[i][j-1] + \text{cost}(\text{insert}) & \text{(insert } y[j]\text{)} \\
c[i-2][j-2] + \text{cost}(\text{twiddle}) & \text{if } x[i-1] = y[j] \text{ and } x[i] = y[j-1]
\end{cases}$$

**Base cases:**
- $$c[0][0] = 0$$ (empty to empty requires no operations)
- $$c[i][0] = i \cdot \text{cost}(\text{delete})$$ (delete all characters from $$x$$)
- $$c[0][j] = j \cdot \text{cost}(\text{insert})$$ (insert all characters into $$z$$)

**Handling Kill:** The kill operation is special because it must be the final operation and it discards the remainder of $$x$$. After computing the full table, we check if using kill is beneficial:

$$\text{Final answer} = \min\left( c[m][n], \min_{i < m} \{ c[i][n] + \text{cost}(\text{kill}) \} \right)$$

{% capture code %}
Edit-Distance(x, y, m, n, cost)
    let c[0..m, 0..n] be a new table
    let op[0..m, 0..n] be a new table // for reconstruction

    // Base cases
    c[0][0] = 0
    for i = 1 to m
        c[i][0] = i * cost[delete]
        op[i][0] = DELETE
    for j = 1 to n
        c[0][j] = j * cost[insert]
        op[0][j] = INSERT

    // Fill the table
    for i = 1 to m
        for j = 1 to n
            c[i][j] = infinity

            // Copy
            if x[i] == y[j] and c[i-1][j-1] + cost[copy] < c[i][j]
                c[i][j] = c[i-1][j-1] + cost[copy]
                op[i][j] = COPY

            // Replace
            if c[i-1][j-1] + cost[replace] < c[i][j]
                c[i][j] = c[i-1][j-1] + cost[replace]
                op[i][j] = REPLACE

            // Delete
            if c[i-1][j] + cost[delete] < c[i][j]
                c[i][j] = c[i-1][j] + cost[delete]
                op[i][j] = DELETE

            // Insert
            if c[i][j-1] + cost[insert] < c[i][j]
                c[i][j] = c[i][j-1] + cost[insert]
                op[i][j] = INSERT

            // Twiddle
            if i >= 2 and j >= 2
                if x[i-1] == y[j] and x[i] == y[j-1]
                    if c[i-2][j-2] + cost[twiddle] < c[i][j]
                        c[i][j] = c[i-2][j-2] + cost[twiddle]
                        op[i][j] = TWIDDLE

    // Check if kill is beneficial
    best = c[m][n]
    kill-point = m + 1
    for i = 0 to m - 1
        if c[i][n] + cost[kill] < best
            best = c[i][n] + cost[kill]
            kill-point = i

    return best, op, kill-point

**Running Time:** We fill an $$m \times n$$ table, and each entry takes $$O(1)$$ time to compute. The total time is $$\Theta(mn)$$.

**Space:** We need $$\Theta(mn)$$ space for the tables $$c$$ and $$op$$. If we only need the distance (not the operation sequence), we can reduce space to $$\Theta(\min(m, n))$$ by keeping only two rows of the table.

### B. DNA Sequence Alignment as Edit Distance

The DNA alignment scoring system assigns:
- $$+1$$ for matching characters
- $$-1$$ for mismatching characters
- $$-2$$ for gaps (spaces)

We want to **maximize** the alignment score, whereas edit distance **minimizes** cost. We can convert between them by negating the values.

**Mapping operations:**

| Alignment Action | Edit Operation | Original Score | Edit Cost |
|------------------|----------------|----------------|-----------|
| Match (no gap, chars equal) | Copy | $$+1$$ | $$-1$$ |
| Mismatch (no gap, chars differ) | Replace | $$-1$$ | $$+1$$ |
| Gap in $$y$$ | Delete | $$-2$$ | $$+2$$ |
| Gap in $$x$$ | Insert | $$-2$$ | $$+2$$ |

The operations **Twiddle** and **Kill** are not used, since:
- Twiddle swaps adjacent characters, which doesn't correspond to any alignment operation
- Kill discards the remainder of $$x$$, but in alignment we must account for all characters in both sequences

With these costs, minimizing edit distance is equivalent to maximizing alignment score (after negating the final value).

{% capture note %}
This connection between sequence alignment and edit distance is fundamental in bioinformatics. The dynamic programming algorithm for sequence alignment (known as the [Needleman-Wunsch algorithm](https://en.wikipedia.org/wiki/Needleman%E2%80%93Wunsch_algorithm){:target="_blank"}) is essentially the same as the edit distance algorithm.
