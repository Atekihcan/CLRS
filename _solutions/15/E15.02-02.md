> Give a recursive algorithm $$\textsc{Matrix-Chain-Multiply}(A, s, i, j)$$ that actually performs the optimal matrix-chain multiplication, given the sequence of matrices $$\langle A_1, A_2, \ldots, A_n \rangle$$, the $$s$$ table computed by $$\textsc{Matrix-Chain-Order}$$, and the indices $$i$$ and $$j$$. (The initial call would be $$\textsc{Matrix-Chain-Multiply}(A, s, 1, n)$$.)

Once we have computed the optimal parenthesization using $$\textsc{Matrix-Chain-Order}$$, the $$s$$ table tells us exactly how to split each subchain. The key insight is that $$s[i,j]$$ gives us the index $$k$$ where we should split the product $$A_i A_{i+1} \cdots A_j$$ into two subproducts: $$A_i \cdots A_k$$ and $$A_{k+1} \cdots A_j$$.

Think of the $$s$$ table as a recipe book: for any subchain you want to compute, it tells you exactly how to break it down into smaller pieces. We recursively follow this recipe until we reach individual matrices.

{% capture code %}
Matrix-Chain-Multiply(A, s, i, j)
    if i == j
        return A[i]
    else
        X = Matrix-Chain-Multiply(A, s, i, s[i, j])
        Y = Matrix-Chain-Multiply(A, s, s[i, j] + 1, j)
        return Matrix-Multiply(X, Y)
{% endcapture %}

The algorithm works as follows:

**Base case**: When $$i = j$$, we have a single matrix $$A_i$$, which we simply return.

**Recursive case**: When $$i < j$$, we:
1. Look up the optimal split point $$k = s[i,j]$$
2. Recursively compute the left subproduct $$A_i \cdots A_k$$
3. Recursively compute the right subproduct $$A_{k+1} \cdots A_j$$
4. Multiply these two resulting matrices using the standard $$\textsc{Matrix-Multiply}$$ algorithm

{% include ads.html %}

The initial call $$\textsc{Matrix-Chain-Multiply}(A, s, 1, n)$$ computes the full product $$A_1 A_2 \cdots A_n$$ using the optimal parenthesization encoded in $$s$$.

{% capture note %}
The algorithm performs exactly $$n - 1$$ matrix multiplications for a chain of $$n$$ matrices (one less than the number of matrices, just like fully parenthesizing an expression requires $$n - 1$$ pairs of parentheses, as shown in Exercise 15.2-6). Each multiplication combines two intermediate results until we have the final product.
{% endcapture %}
