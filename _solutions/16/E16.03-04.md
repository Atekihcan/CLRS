> Prove that we can also express the total cost of a tree for a code as the sum, over all internal nodes, of the combined frequencies of the two children of the node.

The standard definition of the cost of a tree $$T$$ is given by equation (16.4):

$$B(T) = \sum_{c \in C} c.\text{freq} \cdot d_T(c)$$

where $$d_T(c)$$ is the depth of character $$c$$ in the tree. We want to show this equals the sum of merged frequencies at each internal node.

Think about what each character's contribution to $$B(T)$$ really means. A character $$c$$ at depth $$d$$ contributes $$c.\text{freq} \cdot d$$ to the total cost. We can think of this as the character "paying" its frequency once for each edge on the path from the root to its leaf.

Now consider an arbitrary internal node $$v$$ in the tree. Let $$S_v$$ denote the set of characters (leaves) in the subtree rooted at $$v$$. The combined frequency of the children of $$v$$ is exactly $$\sum_{c \in S_v} c.\text{freq}$$, because the frequency stored at $$v$$ is the sum of frequencies of all leaves in its subtree.

Each character in $$S_v$$ has its path from the root pass through node $$v$$. Specifically, the edge entering $$v$$ (from $$v$$'s parent) is traversed exactly once by each character in $$S_v$$. So the contribution of this edge to $$B(T)$$ is $$\sum_{c \in S_v} c.\text{freq}$$, which is precisely the combined frequency stored at node $$v$$.

To make this precise, let's rewrite $$B(T)$$ by counting edge traversals:

$$B(T) = \sum_{c \in C} c.\text{freq} \cdot d_T(c) = \sum_{c \in C} c.\text{freq} \cdot (\text{number of edges on path to } c)$$

We can swap the order of summation by counting each edge's contribution:

$$B(T) = \sum_{\text{edges } e} \sum_{c \text{ whose path uses } e} c.\text{freq}$$

For each edge $$e$$ connecting a parent to its child $$v$$, the characters whose paths use $$e$$ are exactly those in the subtree rooted at $$v$$. The inner sum is thus the combined frequency of the subtree at $$v$$, which equals the frequency value stored at $$v$$.

Since every internal node (except the root) has exactly one incoming edge, and the combined frequency at each internal node equals the sum of frequencies of its children, we have:

$$B(T) = \sum_{\text{internal nodes } v \neq \text{root}} v.\text{freq}$$

But $$v.\text{freq}$$ is defined recursively as the sum of the frequencies of $$v$$'s two children. So we can equivalently write:

$$B(T) = \sum_{\text{internal nodes } v} (v.\text{left}.\text{freq} + v.\text{right}.\text{freq})$$

{% capture note %}
This alternative formulation of tree cost explains why the Huffman algorithm is greedy. Each merge operation creates an internal node, and the cost of that merge is the sum of the two frequencies being merged. The total cost of the tree is the sum of all merge costs. By always choosing to merge the two smallest frequencies, Huffman's algorithm minimizes the cost of each merge, which in turn minimizes the total cost.
