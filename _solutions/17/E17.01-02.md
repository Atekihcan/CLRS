> Show that if a $$\textsc{Decrement}$$ operation were included in the $$k$$-bit counter example, $$n$$ operations could cost as much as $$\Theta(nk)$$ time.

The key to the original $$O(1)$$ amortized analysis for $$\textsc{Increment}$$ is that higher-order bits flip less frequently than lower-order bits. Bit $$A[i]$$ flips only once every $$2^i$$ operations, creating a rapidly decreasing series. Adding $$\textsc{Decrement}$$ destroys this structure by allowing us to repeatedly oscillate between adjacent counter values.

Consider a $$k$$-bit counter initialized to $$0$$, and consider the sequence of alternating operations:

$$\textsc{Increment}, \textsc{Decrement}, \textsc{Increment}, \textsc{Decrement}, \ldots$$

Starting from $$0$$:
- $$\textsc{Increment}$$ changes $$0 \to 1$$: flips 1 bit (cost 1)
- $$\textsc{Decrement}$$ changes $$1 \to 0$$: flips 1 bit (cost 1)

This gives $$O(1)$$ per operation. But now consider starting from $$2^{k-1} - 1$$ (the value with the first $$k-1$$ bits all set to 1):

Starting from binary $$0111\ldots1$$ ($$k-1$$ ones):
- $$\textsc{Increment}$$ changes $$0111\ldots1 \to 1000\ldots0$$: flips all $$k$$ bits (cost $$k$$)
- $$\textsc{Decrement}$$ changes $$1000\ldots0 \to 0111\ldots1$$: flips all $$k$$ bits (cost $$k$$)

{% include ads.html %}

Each operation in this oscillating sequence costs $$\Theta(k)$$ time. If we repeat this pair $$n/2$$ times, the total cost is:

$$\frac{n}{2} \cdot 2k = nk = \Theta(nk)$$

This demonstrates that with $$\textsc{Decrement}$$, we can force a sequence of $$n$$ operations to cost $$\Theta(nk)$$, which means the amortized cost per operation is $$\Theta(k)$$, not $$O(1)$$.

{% capture note %}
The aggregate analysis for $$\textsc{Increment}$$ alone exploits the asymmetry of counting up: bit $$i$$ can only flip from 1 to 0 when bit $$i-1$$ overflows from 1 to 0. With $$\textsc{Decrement}$$, we introduce a symmetric "borrow" operation that lets us cycle between states indefinitely, negating the telescoping effect that made the original analysis work.
