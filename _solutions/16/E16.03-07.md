> Generalize Huffman's algorithm to ternary codewords (i.e., codewords using the symbols 0, 1, and 2), and prove that it yields optimal ternary codes.

A ternary Huffman code uses three symbols (0, 1, 2) instead of two (0, 1). The code is represented by a full ternary tree where each internal node has exactly three children. The key insight is that Huffman's greedy strategy, which merges minimum-frequency elements, extends naturally to this setting.

### Ternary Huffman Algorithm

The algorithm is similar to binary Huffman, but we merge three nodes at a time instead of two:

{% capture code %}
Ternary-Huffman(C)
    n = |C|
    // Pad with dummy zero-frequency characters if needed
    while (n - 1) mod 2 ≠ 0
        add a dummy character with frequency 0 to C
        n = n + 1
    Q = C
    while |Q| > 1
        allocate a new node z
        z.left = x = Extract-Min(Q)
        z.middle = y = Extract-Min(Q)
        z.right = w = Extract-Min(Q)
        z.freq = x.freq + y.freq + w.freq
        Insert(Q, z)
    return Extract-Min(Q)
{% endcapture %}

The padding step ensures we can always merge exactly three nodes at each step until we reach the root. This is necessary because a full ternary tree with $$n$$ leaves has internal nodes that account for exactly $$n - 1$$ edges to non-root nodes, divided among ternary splits.

### Why Padding is Necessary

In a full ternary tree, each internal node has 3 children, contributing 3 edges. If there are $$k$$ internal nodes, there are $$3k$$ edges, and thus $$3k + 1$$ nodes total (each non-root node has one incoming edge). With $$n$$ leaves, we have $$n + k = 3k + 1$$, so $$k = (n-1)/2$$. This means $$(n - 1)$$ must be even for a full ternary tree to exist.

If $$n \equiv 0 \pmod{2}$$ (i.e., $$n - 1$$ is odd), we add one dummy character. If $$n \equiv 1 \pmod{2}$$ and $$n \not\equiv 1 \pmod{2}$$ after adjustment... Actually, let's be more precise: we need $$(n - 1) \bmod 2 = 0$$, which means $$n$$ must be odd. So:
- If $$n$$ is odd: no padding needed
- If $$n$$ is even: add one dummy character with frequency 0

### Proof of Optimality

The proof follows the same structure as for binary Huffman codes:

**Lemma (Greedy Choice):** Let $$x$$, $$y$$, and $$z$$ be three characters with the lowest frequencies. There exists an optimal ternary prefix code where $$x$$, $$y$$, and $$z$$ are siblings at the maximum depth.

The proof mirrors Lemma 16.2. Take any optimal tree $$T$$ and let $$a$$, $$b$$, $$c$$ be three siblings at maximum depth. By swapping $$x$$, $$y$$, $$z$$ with $$a$$, $$b$$, $$c$$ (similar to the binary case), we can show the cost does not increase, since we're placing the lowest-frequency characters at the deepest level.

**Lemma (Optimal Substructure):** If we merge $$x$$, $$y$$, $$z$$ into a new character $$w$$ with frequency $$x.\text{freq} + y.\text{freq} + z.\text{freq}$$, then an optimal tree for the reduced alphabet plus $$w$$ gives an optimal tree for the original alphabet when we expand $$w$$ back into its three children.

The proof follows Lemma 16.3: if $$T$$ is optimal for the reduced alphabet, and we replace the leaf for $$w$$ with an internal node having $$x$$, $$y$$, $$z$$ as children, the resulting tree $$T'$$ is optimal for the original alphabet.

**Theorem:** The ternary Huffman algorithm produces an optimal ternary prefix code.

This follows immediately from the two lemmas, just as in the binary case.

{% capture note %}
This generalization extends to any radix $$r \ge 2$$. For an $$r$$-ary Huffman code, we merge $$r$$ minimum-frequency characters at each step and may need to pad with up to $$r - 2$$ dummy characters to ensure $$(n - 1) \bmod (r - 1) = 0$$.
{% endcapture %}
