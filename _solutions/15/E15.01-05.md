> The Fibonacci numbers are defined by recurrence (3.22). Give an $$O(n)$$-time dynamic-programming algorithm to compute the $$n$$th Fibonacci number. Draw the subproblem graph. How many vertices and edges are in the graph?

The Fibonacci sequence is defined by the recurrence:

$$F_n = \begin{cases} 0 & \text{if } n = 0 \\ 1 & \text{if } n = 1 \\ F_{n-1} + F_{n-2} & \text{if } n \ge 2 \end{cases}$$

A naive recursive implementation would be exponential because it recomputes the same subproblems many times. For example, computing $$F_5$$ requires $$F_4$$ and $$F_3$$, but computing $$F_4$$ also requires $$F_3$$, leading to redundant work. Dynamic programming eliminates this redundancy by computing each Fibonacci number exactly once.

Here is the bottom-up dynamic programming algorithm:

{% capture code %}
Fibonacci(n)
    if n == 0
        return 0
    let F[0..n] be a new array
    F[0] = 0
    F[1] = 1
    for i = 2 to n
        F[i] = F[i - 1] + F[i - 2]
    return F[n]
{% endcapture %}
{% include clrs_code.html title="Fibonacci(n)" %}

The algorithm fills in the array from the base cases up to $$F_n$$, using previously computed values. Each value is computed in $$O(1)$$ time (a single addition), and we compute $$n - 1$$ values, giving a total running time of $$O(n)$$.

**The Subproblem Graph**

The subproblem graph has one vertex for each subproblem $$F_i$$ where $$i \in \{0, 1, 2, \ldots, n\}$$. There is a directed edge from vertex $$i$$ to vertex $$j$$ if computing $$F_i$$ directly requires the value of $$F_j$$.

For $$i \ge 2$$, computing $$F_i$$ requires $$F_{i-1}$$ and $$F_{i-2}$$, so there are edges from $$i$$ to $$i-1$$ and from $$i$$ to $$i-2$$. The base cases $$F_0$$ and $$F_1$$ have no outgoing edges.

The subproblem graph for $$n = 5$$ looks like this:

$$5 \rightarrow 4 \rightarrow 3 \rightarrow 2 \rightarrow 1$$

with additional edges: $$5 \rightarrow 3$$, $$4 \rightarrow 2$$, $$3 \rightarrow 1$$, $$2 \rightarrow 0$$.

**Counting Vertices and Edges**

The graph has exactly $$n + 1$$ vertices, one for each of $$F_0, F_1, \ldots, F_n$$.

For edges:
- Vertices 0 and 1 have 0 outgoing edges (base cases)
- Vertices 2 through $$n$$ each have exactly 2 outgoing edges

Therefore, the number of edges is $$2(n - 1) = 2n - 2$$ for $$n \ge 2$$.

{% capture note %}
The subproblem graph for Fibonacci is remarkably simple compared to rod cutting. In rod cutting with $$n = 4$$, vertex 4 has edges to vertices 0, 1, 2, and 3 (four edges). In Fibonacci with $$n = 4$$, vertex 4 has edges only to vertices 3 and 2 (two edges). This structural difference explains why Fibonacci can be solved in $$O(n)$$ time while rod cutting requires $$O(n^2)$$ time. The number of edges in the subproblem graph often determines the algorithm's running time.
{% endcapture %}
{% include aside.html title='Why linear time?' %}

{% capture note %}
The algorithm above uses $$O(n)$$ space for the array. Since each $$F_i$$ only depends on $$F_{i-1}$$ and $$F_{i-2}$$, we can reduce space to $$O(1)$$ by keeping only the last two values:

```
prev2, prev1 = 0, 1
for i = 2 to n:
    curr = prev1 + prev2
    prev2, prev1 = prev1, curr
return prev1
```

This is a common optimization when the dependency structure allows it.
{% endcapture %}
