> Consider a modification of the rod-cutting problem in which, in addition to a price $$p_i$$ for each rod, each cut incurs a fixed cost of $$c$$. The revenue associated with a solution is now the sum of the prices of the pieces minus the costs of making the cuts. Give a dynamic-programming algorithm to solve this modified problem.

In the original problem, cuts are free, so we focus only on maximizing the total price of pieces. Now, each cut costs $$c$$ dollars, which changes the trade-off: making more cuts might give us better piece prices, but we pay for each cut. Think of it like a lumber mill where operating the saw has a cost.

The key insight is that when we make a cut to separate a piece of length $$i$$ from a rod of length $$n$$, we incur cost $$c$$ (unless $$i = n$$, meaning we sell the rod uncut). The remaining piece of length $$n - i$$ will be solved optimally by our subproblem, which already accounts for any cuts made within it.

Let $$r_n$$ denote the maximum revenue obtainable from a rod of length $$n$$. The modified recurrence becomes:

$$r_n = \max_{1 \le i \le n} \begin{cases} p_i + r_{n-i} - c & \text{if } i < n \text{ (we make a cut)} \\ p_n & \text{if } i = n \text{ (no cut)} \end{cases}$$

We can simplify this by handling the no-cut case separately:

$$r_n = \max\left(p_n, \max_{1 \le i < n} (p_i + r_{n-i} - c)\right)$$

Alternatively, we can write it more uniformly as:

$$r_n = \max_{1 \le i \le n} (p_i + r_{n-i} - c \cdot \mathbf{1}_{i < n})$$

where $$\mathbf{1}_{i < n}$$ is 1 if we make a cut ($$i < n$$) and 0 otherwise.

Here is the bottom-up dynamic programming algorithm:

{% capture code %}
Modified-Bottom-Up-Cut-Rod(p, n, c)
    let r[0..n] be a new array
    r[0] = 0
    for j = 1 to n
        q = p[j]                        // no cut: sell rod of length j as-is
        for i = 1 to j - 1
            q = max(q, p[i] + r[j - i] - c)
        r[j] = q
    return r[n]
{% endcapture %}
{% include clrs_code.html title="Modified-Bottom-Up-Cut-Rod(p, n, c)" %}

The algorithm initializes $$q$$ to $$p_j$$ (the price for selling the rod uncut), then considers all ways to cut off a first piece of length $$i < j$$, subtracting the cut cost $$c$$ from each option. This ensures we only pay for cuts we actually make.

The running time remains $$\Theta(n^2)$$, the same as the original algorithm, since we still have the same nested loop structure.

{% capture note %}
When the cut cost $$c$$ is very high, the optimal solution tends toward fewer cuts or no cuts at all. When $$c = 0$$, we recover the original rod-cutting problem. This modification models real-world scenarios where setup costs, machine time, or material waste make each cut expensive.
