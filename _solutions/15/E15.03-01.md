> Which is a more efficient way to determine the optimal number of multiplications in a matrix-chain multiplication problem: enumerating all the ways of parenthesizing the product and computing the number of multiplications for each, or running $$\textsc{Recursive-Matrix-Chain}$$? Justify your answer.

Running $$\textsc{Recursive-Matrix-Chain}$$ is more efficient than enumerating all parenthesizations, though both approaches are exponential and impractical for large inputs.

To understand why, consider what each approach actually computes. When we enumerate all parenthesizations, we explicitly generate each of the $$P(n)$$ different ways to parenthesize $$n$$ matrices, where $$P(n)$$ is the number of full parenthesizations. For each parenthesization, we compute the number of scalar multiplications required, which takes $$\Theta(n)$$ time since we need to traverse the parenthesization structure. The total work is therefore $$\Theta(n \cdot P(n))$$.

In Section 15.2, the book shows that $$P(n)$$ equals the $$(n-1)$$th Catalan number:

$$P(n) = C(n-1) = \frac{1}{n} \binom{2n-2}{n-1} = \Omega\left(\frac{4^n}{n^{3/2}}\right)$$

So the enumeration approach runs in $$\Omega(4^n / n^{1/2})$$ time.

Now consider $$\textsc{Recursive-Matrix-Chain}$$. As analyzed in Section 15.3, this recursive algorithm runs in $$\Omega(2^n)$$ time. The key insight is that while it does redundant work by solving the same subproblems multiple times, it never explicitly enumerates all parenthesizations. Instead, it explores the subproblem space, and although this space overlaps heavily (leading to exponential time), the base of the exponent is 2 rather than 4.

Since $$2^n$$ grows asymptotically slower than $$4^n/n^{3/2}$$ (because $$4^n = 2^{2n}$$), $$\textsc{Recursive-Matrix-Chain}$$ is more efficient despite both approaches being impractically slow.

{% capture note %}
Neither approach is practical for real applications. The dynamic programming algorithm $$\textsc{Matrix-Chain-Order}$$ runs in just $$\Theta(n^3)$$ time by eliminating redundant computation through memoization. This transforms an exponential-time problem into a polynomial-time one.
