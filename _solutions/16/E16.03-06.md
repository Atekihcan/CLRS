> Suppose we have an optimal prefix code on a set $$C = \{0, 1, \ldots, n - 1\}$$ of characters and we wish to transmit this code using as few bits as possible. Show how to represent any optimal prefix code on $$C$$ using only $$2n - 1 + n\lceil \lg n \rceil$$ bits. (Hint: Use $$2n - 1$$ bits to specify the structure of the tree, as discovered by a walk of the tree.)

To transmit a prefix code, we need to communicate two pieces of information: the shape of the binary tree, and which character is assigned to which leaf. Let's handle each part separately.

### Part 1: Tree Structure ($$2n - 1$$ bits)

An optimal prefix code is represented by a full binary tree with $$n$$ leaves (one for each character). By Exercise 16.3-2, we know optimal codes correspond to full binary trees. A full binary tree with $$n$$ leaves has exactly $$n - 1$$ internal nodes, for a total of $$2n - 1$$ nodes.

We can encode the tree structure using a preorder traversal (visit root, then left subtree, then right subtree). During the traversal, output:
- 0 for each internal node
- 1 for each leaf node

This produces a string of exactly $$2n - 1$$ bits: $$(n - 1)$$ zeros for the internal nodes and $$n$$ ones for the leaves.

For example, a tree with structure:
```
      *
     / \
    *   c
   / \
  a   b
```
would be encoded as: 0, 0, 1, 1, 1 (preorder: root(internal), left child(internal), a(leaf), b(leaf), c(leaf)).

This encoding uniquely determines the tree structure. To decode, we can reconstruct the tree by reading bits left to right: a 0 means "create an internal node and recurse on left and right children," and a 1 means "create a leaf."

### Part 2: Character Assignments ($$n\lceil \lg n \rceil$$ bits)

We have $$n$$ characters $$\{0, 1, \ldots, n - 1\}$$ to assign to $$n$$ leaves. We need to specify which character goes in each leaf, in the order the leaves are encountered during the traversal.

Each character can be represented using $$\lceil \lg n \rceil$$ bits (since we need to distinguish among $$n$$ possibilities). With $$n$$ leaves, we need $$n \cdot \lceil \lg n \rceil$$ bits total for the character assignments.

### Total Bits

Combining both parts:

$$\text{Total bits} = (2n - 1) + n\lceil \lg n \rceil$$

This matches exactly what the problem asks us to show.

{% capture note %}
This representation is quite efficient. For the tree structure, we're using what's known as a succinct representation: the information-theoretic minimum for representing a binary tree with $$n$$ leaves is $$\Theta(n)$$ bits, and we're using exactly $$2n - 1$$. The character assignment portion could potentially be compressed further using the fact that it's a permutation (which has $$n!$$ possibilities, requiring $$\lceil \lg(n!) \rceil \approx n \lg n - n \lg e$$ bits), but the simple approach of $$n \lceil \lg n \rceil$$ bits is clean and nearly optimal.
{% endcapture %}
