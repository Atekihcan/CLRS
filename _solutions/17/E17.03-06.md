> Show how to implement a queue with two ordinary stacks (Exercise 10.1-6) so that the amortized cost of each $$\textsc{Enqueue}$$ and each $$\textsc{Dequeue}$$ operation is $$O(1)$$.

A queue processes elements in first-in, first-out (FIFO) order, while a stack processes them in last-in, first-out (LIFO) order. The clever insight is that passing elements through two stacks reverses their order twice, which restores the original order. Think of it like pouring water from one cup to another and back: the order reverses each time, so two transfers restore the original order.

We maintain two stacks: an "in-stack" $$S_{in}$$ for enqueuing and an "out-stack" $$S_{out}$$ for dequeuing.

**$$\textsc{Enqueue}(x)$$:** Push $$x$$ onto $$S_{in}$$. Cost: $$O(1)$$.

**$$\textsc{Dequeue}$$:** If $$S_{out}$$ is non-empty, pop from $$S_{out}$$ and return it. Otherwise, transfer all elements from $$S_{in}$$ to $$S_{out}$$ (by repeatedly popping from $$S_{in}$$ and pushing onto $$S_{out}$$), then pop from $$S_{out}$$. The transfer reverses the order, so the oldest element in $$S_{in}$$ becomes the top of $$S_{out}$$.

The worst-case cost of $$\textsc{Dequeue}$$ can be $$O(n)$$ when we need to transfer all elements, but the amortized cost is $$O(1)$$. To prove this, we use the potential method.

Define the potential function:

$$\Phi = |S_{in}|$$

In other words, the potential equals the number of elements in the in-stack. Initially, both stacks are empty, so $$\Phi(D_0) = 0$$. Since $$|S_{in}| \ge 0$$ always, we have $$\Phi(D_i) \ge 0$$.

**Amortized cost of $$\textsc{Enqueue}$$:**

Pushing onto $$S_{in}$$ costs 1 and increases $$|S_{in}|$$ by 1.

$$\hat{c}_{\textsc{Enqueue}} = 1 + (|S_{in}| + 1) - |S_{in}| = 2$$

**Amortized cost of $$\textsc{Dequeue}$$:**

If $$S_{out}$$ is non-empty, we simply pop from it. The cost is 1, and $$|S_{in}|$$ does not change.

$$\hat{c}_{\textsc{Dequeue}} = 1 + 0 = 1$$

If $$S_{out}$$ is empty, we transfer all $$k = |S_{in}|$$ elements. Each transfer involves one pop from $$S_{in}$$ and one push to $$S_{out}$$, costing 2 per element, plus one final pop from $$S_{out}$$. The actual cost is $$2k + 1$$, and $$|S_{in}|$$ drops from $$k$$ to 0.

$$\hat{c}_{\textsc{Dequeue}} = (2k + 1) + (0 - k) = k + 1$$

Wait, this seems too high. Let us reconsider. The cost of transferring $$k$$ elements is $$k$$ pops + $$k$$ pushes = $$2k$$. But each push and pop is $$O(1)$$, so we can count them as 1 each. Using the simpler model where the transfer costs $$k$$ (for $$k$$ pops) plus $$k$$ (for $$k$$ pushes), the final pop costs 1:

$$\hat{c}_{\textsc{Dequeue}} = (2k + 1) + (0 - k) = k + 1$$

Hmm, this is not $$O(1)$$. Let us redefine the potential to be $$\Phi = 2|S_{in}|$$:

**Amortized cost of $$\textsc{Enqueue}$$:**

$$\hat{c}_{\textsc{Enqueue}} = 1 + 2(|S_{in}| + 1) - 2|S_{in}| = 3$$

**Amortized cost of $$\textsc{Dequeue}$$ (with transfer):**

$$\hat{c}_{\textsc{Dequeue}} = (2k + 1) + (0 - 2k) = 1$$

With potential $$\Phi = 2|S_{in}|$$, both operations have amortized cost $$O(1)$$.
