> Not just any greedy approach to the activity-selection problem produces a maximum-size set of mutually compatible activities. Give an example to show that the approach of selecting the activity of least duration from among those that are compatible with previously selected activities does not work. Do the same for the approaches of always selecting the compatible activity that overlaps the fewest other remaining activities and always selecting the compatible remaining activity with the earliest start time.

This exercise teaches us that being greedy is not enough; we must be greedy about the right thing. The earliest-finish-time strategy works because it maximizes the remaining time for other activities. Other seemingly reasonable greedy choices fail because they optimize the wrong quantity.

### Shortest Duration Fails

Consider three activities:

| Activity | Start | Finish | Duration |
|----------|-------|--------|----------|
| $$a_1$$  | 0     | 4      | 4        |
| $$a_2$$  | 3     | 5      | 2        |
| $$a_3$$  | 4     | 8      | 4        |

The shortest-duration greedy selects $$a_2$$ first (duration 2). After selecting $$a_2$$, neither $$a_1$$ nor $$a_3$$ is compatible with it. So the greedy solution is $$\{a_2\}$$ with size 1.

However, the optimal solution is $$\{a_1, a_3\}$$ with size 2. Activity $$a_1$$ finishes at time 4, exactly when $$a_3$$ starts, so they are compatible.

The short activity $$a_2$$ acts as a "blocker" that sits in the middle and prevents us from taking both longer activities on either side.

### Fewest Conflicts Fails

Consider these activities:

| Activity | Start | Finish | Conflicts With |
|----------|-------|--------|----------------|
| $$a_1$$  | 0     | 2      | $$a_3$$        |
| $$a_2$$  | 1     | 3      | $$a_3, a_4$$   |
| $$a_3$$  | 1     | 3      | $$a_1, a_2, a_4, a_5$$ |
| $$a_4$$  | 2     | 4      | $$a_2, a_3$$   |
| $$a_5$$  | 3     | 5      | $$a_3$$        |

Count the conflicts: $$a_1$$ has 1, $$a_2$$ has 2, $$a_3$$ has 4, $$a_4$$ has 2, $$a_5$$ has 1.

The fewest-conflicts greedy picks either $$a_1$$ or $$a_5$$ first (both have 1 conflict). Suppose we pick $$a_1$$. After removing $$a_1$$ and incompatible activities, we recount: $$a_4$$ and $$a_5$$ remain compatible, both with 0 conflicts now. We pick $$a_4$$, then $$a_5$$. Result: $$\{a_1, a_4, a_5\}$$ with size 3? Actually, let's verify: $$a_4$$ is $$[2,4)$$ and $$a_5$$ is $$[3,5)$$, they overlap! So we get $$\{a_1, a_5\}$$ with size 2.

Actually, let me provide a cleaner counterexample. Consider:

| Activity | Start | Finish |
|----------|-------|--------|
| $$a_1$$  | 0     | 4      |
| $$a_2$$  | 2     | 6      |
| $$a_3$$  | 4     | 8      |
| $$a_4$$  | 6     | 10     |
| $$a_5$$  | 8     | 12     |
| $$a_6$$  | 4     | 6      |

Activity $$a_6$$ conflicts with $$a_1, a_2, a_3, a_4$$, giving it 4 conflicts. Activities $$a_1, a_5$$ each have 1 conflict. The fewest-conflicts approach might select $$a_1$$, then $$a_3$$ or $$a_5$$. But $$a_6$$ is key to a better solution: the optimal is $$\{a_1, a_6, a_5\}$$ with size 3.

The simplest counterexample uses this structure: one short activity in the middle overlaps many others, so it has high conflict count, but selecting it enables more total activities.

### Earliest Start Time Fails

Consider three activities:

| Activity | Start | Finish |
|----------|-------|--------|
| $$a_1$$  | 0     | 10     |
| $$a_2$$  | 1     | 3      |
| $$a_3$$  | 4     | 6      |

The earliest-start greedy selects $$a_1$$ first (starts at 0). After selecting $$a_1$$, neither $$a_2$$ nor $$a_3$$ is compatible. So the greedy solution is $$\{a_1\}$$ with size 1.

However, the optimal solution is $$\{a_2, a_3\}$$ with size 2. Both $$a_2$$ and $$a_3$$ fit entirely within the interval where $$a_1$$ runs.

This example clearly shows the flaw: an activity that starts early but runs for a long time can block multiple shorter activities.

{% capture note %}
The common thread in all these failures is that the greedy choice optimizes a local property (short duration, few conflicts, early start) that doesn't necessarily lead to the global optimum (maximum number of activities). Only the earliest-finish-time criterion directly addresses leaving maximum time available for remaining activities, which is what we truly need to maximize.
