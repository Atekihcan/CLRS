> Show how to compute the length of an LCS using only $$2 \cdot \min(m, n)$$ entries in the $$c$$ table plus $$O(1)$$ additional space. Then show how to do the same thing, but using $$\min(m, n)$$ entries plus $$O(1)$$ additional space.

When we examine the recurrence for computing $$c[i, j]$$, we notice that each entry depends only on three other entries: $$c[i-1, j-1]$$, $$c[i-1, j]$$, and $$c[i, j-1]$$. All of these are either in the current row or the immediately previous row. This observation is the key to reducing space.

### Using $$2 \cdot \min(m, n)$$ Entries

First, let's ensure we're iterating over the shorter sequence to minimize space. If $$n < m$$, we swap $$X$$ and $$Y$$ (this doesn't change the LCS length). Now we can assume $$n \leq m$$ without loss of generality.

Since each row only depends on the previous row, we only need to keep two rows at any time: the current row we're computing and the previous row we're reading from. Each row has $$n + 1$$ entries, giving us $$2(n + 1) = O(\min(m, n))$$ space.

{% capture code %}
LCS-Length-Two-Rows(X, Y)
    m = X.length
    n = Y.length
    if m < n
        swap X and Y
        swap m and n
    let prev[0..n] and curr[0..n] be new arrays
    for j = 0 to n
        prev[j] = 0
    for i = 1 to m
        curr[0] = 0
        for j = 1 to n
            if x_i == y_j
                curr[j] = prev[j - 1] + 1
            else
                curr[j] = max(prev[j], curr[j - 1])
        swap prev and curr
    return prev[n]
{% endcapture %}

### Using $$\min(m, n)$$ Entries

We can do even better by observing that within a single row, we only need to remember one value from the previous row at any moment: the diagonal element $$c[i-1, j-1]$$. As we sweep left to right, the "previous row's value at position $$j$$" becomes the "previous row's value at position $$j-1$$" for the next iteration.

The trick is to save the value that's about to be overwritten before we overwrite it. This saved value will be the diagonal element for the next column.

{% capture code %}
LCS-Length-One-Row(X, Y)
    m = X.length
    n = Y.length
    if m < n
        swap X and Y
        swap m and n
    let c[0..n] be a new array
    for j = 0 to n
        c[j] = 0
    for i = 1 to m
        prev_diag = 0
        for j = 1 to n
            temp = c[j]
            if x_i == y_j
                c[j] = prev_diag + 1
            else
                c[j] = max(c[j], c[j - 1])
            prev_diag = temp
    return c[n]
{% endcapture %}
{% include clrs_code.html title="LCS-Length-One-Row(X, Y)" %}

In this version, `c[j]` initially holds $$c[i-1, j]$$ (from the previous row) and gets updated to $$c[i, j]$$. The variable `prev_diag` holds $$c[i-1, j-1]$$ which we saved before overwriting $$c[j-1]$$.

{% capture note %}
This space optimization comes at a cost: we can only compute the **length** of the LCS, not reconstruct the LCS itself. To reconstruct the actual subsequence, we need to keep enough information to trace back through our decisions, which requires either the full table or the $$b$$ table. The textbook mentions that we can reconstruct in $$O(m + n)$$ time if we keep the full $$c$$ table, but with only one row, we've lost the history of how we got to each cell.
{% endcapture %}
