> Give a dynamic-programming algorithm for the activity-selection problem, based on recurrence (16.2). Have your algorithm compute the sizes $$c[i, j]$$ as defined above and also produce the maximum-size subset of mutually compatible activities.
>
> Assume that the inputs have been sorted as in equation (16.1). Compare the running time of your solution to the running time of $$\textsc{Greedy-Activity-Selector}$$.

The activity-selection problem asks us to find the maximum number of non-overlapping activities. While the greedy approach works beautifully here, let's explore the dynamic programming solution to understand the relationship between these two paradigms.

Recall that $$S_{ij}$$ denotes the set of activities that start after activity $$a_i$$ finishes and finish before activity $$a_j$$ starts. The recurrence from the text is:

$$c[i, j] = \begin{cases} 0 & \text{if } S_{ij} = \emptyset \\ \max_{a_k \in S_{ij}} \{ c[i, k] + c[k, j] + 1 \} & \text{if } S_{ij} \neq \emptyset \end{cases}$$

To make this work cleanly, we add fictitious activities $$a_0$$ with $$f_0 = 0$$ and $$a_{n+1}$$ with $$s_{n+1} = \infty$$. This way, $$S_{0, n+1}$$ contains all real activities, and we seek $$c[0, n+1]$$.

{% include ads.html %}

The key observation is that $$a_k \in S_{ij}$$ if and only if $$f_i \le s_k$$ and $$f_k \le s_j$$. Since activities are sorted by finish time, we fill in the table by increasing values of $$j - i$$.

{% capture code %}
DP-Activity-Selector(s, f)
    n = s.length
    let c[0..n+1, 0..n+1] and act[0..n+1, 0..n+1] be new tables
    for i = 0 to n
        c[i, i+1] = 0                          // no activities between adjacent
    for l = 2 to n + 1                         // l = j - i, the gap size
        for i = 0 to n + 1 - l
            j = i + l
            c[i, j] = 0
            act[i, j] = NIL
            for k = i + 1 to j - 1             // consider each activity in S_ij
                if f[i] <= s[k] and f[k] <= s[j]
                    if c[i, k] + c[k, j] + 1 > c[i, j]
                        c[i, j] = c[i, k] + c[k, j] + 1
                        act[i, j] = k
    return c and act
{% endcapture %}

The running time of this dynamic programming solution is $$\Theta(n^3)$$. We have $$O(n^2)$$ subproblems (pairs $$(i, j)$$), and for each subproblem, we examine up to $$O(n)$$ candidate activities $$a_k$$. The space complexity is also $$O(n^2)$$ for the two tables.

In contrast, $$\textsc{Greedy-Activity-Selector}$$ runs in $$\Theta(n)$$ time (assuming activities are pre-sorted by finish time). This dramatic difference highlights why greedy algorithms are preferred when applicable: the dynamic programming solution does unnecessary work by considering all possible first choices, when in fact the greedy choice (earliest finish time) always works.

{% capture note %}
This exercise illustrates an important principle: dynamic programming is more general but can be overkill. The greedy algorithm is $$n^2$$ times faster because it exploits the special structure of the problem (the greedy-choice property), which lets us commit to one choice without exploring alternatives.
